Authentication

OpenFactory supports two authentication methods: JWT tokens for user sessions and API keys for programmatic / AI-agent access.

JWT Token Flow

For buyers, factories, and admins. Tokens expire after 7 days.

# Register a buyer account
curl -X POST "$BASE_URL/auth/register" \
  -H "Content-Type: application/json" \
  -d '{"email":"buyer@example.com","password":"securepassword","role":"buyer"}'

# Response
{ "user_id": "usr-a1b2c3", "token": "eyJ..." }

# Login
curl -X POST "$BASE_URL/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"email":"buyer@example.com","password":"securepassword"}'

# Response
{ "user_id": "usr-a1b2c3", "role": "buyer", "token": "eyJ..." }

API Key Flow

For AI agents and partner integrations. Keys are prefixed ofk_ and rate-limited per minute.

# Generate key (admin only)
curl -X POST "$BASE_URL/admin/api-keys" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"partner_name":"my-agent","permissions":["GET /factories*","POST /quotes"],"rate_limit_per_min":60}'

# Response
{ "key": "ofk_a1b2c3...", "id": "apk-..." }

# Use in requests
curl -H "Authorization: Bearer ofk_a1b2c3..." "$BASE_URL/factories"
Auth on endpoint cards: Auth = requires Bearer token (JWT or API key). Public = no authentication required.

Base URL

# All endpoints are relative to your deployment origin
BASE_URL="https://your-openfactory-instance.com"

# In browser code, use window.location.origin
# Never hard-code localhost
Rate limits: 100 requests/minute per IP. Auth routes have stricter limits.

Error Handling

All errors return JSON with a consistent shape:

{
  "error": "Human-readable error message"
}
StatusMeaning
400Bad request — invalid input or missing required fields
401Unauthorized — missing or invalid Bearer token
403Forbidden — insufficient permissions
404Not found — resource doesn't exist
409Conflict — invalid state transition (e.g. escrow already locked)
422Unprocessable — invalid status transition
429Rate limited — 100 req/min exceeded

Stage 1 — Factory Discovery

GET /factories Auth

Search and filter verified factories. Returns full factory profiles including contact info, certifications, and trust scores.

Query Parameters
NameTypeDescription
categorystringFilter by product category (e.g. pcb_assembly, plastic_injection)
max_moqnumberMaximum MOQ filter
verified_onlybooleanOnly return USCC-verified factories
min_trust_scorenumberMinimum trust score (0-100)
min_ratingnumberMinimum star rating (1-5)
price_tierstringbudget, mid, or premium
sortstringSort field (e.g. trust_score)
Example
curl -H "Authorization: Bearer $TOKEN" \

Identity trust data for buyer due diligence. Returns USCC code, legal representative, license expiry, and a direct link to China's national enterprise registry (gsxt.gov.cn).

Example

Response
{

  "verified": true,

POST /factories/:id/verify-uscc Public

Validate and store a factory's USCC. Must be 18 alphanumeric characters following the GB 32100-2015 standard.

Request Body
FieldTypeRequiredDescription

curl -X POST "$BASE_URL/factories/sz-006/verify-uscc" \

  "metadata": {

Stage 3 — Quoting

POST /quotes Auth

Request a price quote from a specific factory. Returns unit price, total, lead time, MOQ, and a 7-day valid quote_id for ordering.

Request Body
FieldTypeRequiredDescription
factory_idstringYesFactory ID (e.g. "sz-006")
product_descriptionstringYesDescription of the product
quantitynumberYesNumber of units
buyer_idstringNoBuyer identifier
target_price_usdnumberNoTarget unit price in USD
deadline_daysnumberNoDesired lead time in days
Example
curl -X POST "$BASE_URL/quotes" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "factory_id": "sz-006",
    "product_description": "Custom PCB board, 4-layer, 50x30mm",
    "quantity": 5000,
    "target_price_usd": 3.50
  }'
Response
{
  "quote_id": "q-a1b2c3d4",
  "factory_id": "sz-006",
  "unit_price_usd": 2.38,
  "total_price_usd": 11900.00,
  "lead_time_days": 21,
  "moq": 100,
  "valid_until": "2026-04-02T12:00:00.000Z",
  "notes": "Quote for 5000x Custom PCB board, 4-layer, 50x30mm"
}

POST /quotes/:id/respond Public

Factory responds to a quote request with pricing and lead time. Updates the existing quote record.

Request Body
FieldTypeRequiredDescription
factory_idstringYesFactory ID responding to the quote
unit_price_usdnumberYesUnit price in USD
lead_time_daysnumberYesLead time in days
notesstringNoOptional notes
Example
curl -X POST "$BASE_URL/quotes/q-a1b2c3d4/respond" \
  -H "Content-Type: application/json" \
  -d '{
    "factory_id": "sz-006",
    "unit_price_usd": 2.20,
    "lead_time_days": 18,
    "notes": "Can expedite for additional 15%"

Response
{
  "quote_id": "q-a1b2c3d4",
  "status": "responded",
  "unit_price_usd": 2.20,
  "lead_time_days": 18
}

Stage 4 — Order Execution

POST /orders Auth

Place an order from an accepted quote. Payment is held in escrow. Returns the order_id for tracking.

Request Body
FieldTypeRequiredDescription
quote_idstringYesThe quote_id from POST /quotes or instant-quote
buyer_idstringYesBuyer identifier
Example
curl -X POST "$BASE_URL/orders" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"quote_id": "q-a1b2c3d4", "buyer_id": "buyer-001"}'
Response
{
  "order_id": "ord-e5f6g7h8",
  "quote_id": "q-a1b2c3d4",
  "factory_id": "sz-006",
  "buyer_id": "buyer-001",
  "status": "pending",
  "quantity": 5000,
  "unit_price_usd": 2.38,
  "total_price_usd": 11900.00,
  "escrow_held": true,

POST /orders/:id/milestones Auth

Request Body
FieldTypeRequiredDescription

Example
curl -X POST "$BASE_URL/orders/ord-e5f6g7h8/milestones" \

Response 201

Stage 5 — Escrow

Escrow State Machine
pending_deposit→ deposit_held→ production_released→ qc_released→ final_released

POST /orders/:id/escrow/lock Auth

Lock 30% deposit on an order. Transitions escrow from pending_deposit to deposit_held.

Example
curl -X POST "$BASE_URL/orders/ord-e5f6g7h8/escrow/lock" \
  -H "Authorization: Bearer $TOKEN"
Response
{
  "order_id": "ord-e5f6g7h8",
  "deposit_locked": true,
  "deposit_amount_usd": 3570.00,
  "remaining_usd": 8330.00,
  "escrow_status": "deposit_held",
  "message": "Deposit locked. Factory can now begin production."
}
Idempotency: Returns 409 if escrow is already past pending_deposit state.

POST /orders/:id/escrow/release Auth

Milestone-based escrow release. Validates milestone completion and photo evidence before allowing each transition.

Request Body
FieldTypeRequiredDescription
milestonestringYesproduction_started | qc_pass | shipped
verified_bystringNoWho verified the milestone (e.g. QC inspector ID)
Example
curl -X POST "$BASE_URL/orders/ord-e5f6g7h8/escrow/release" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"milestone": "production_started"}'
Response
{
  "order_id": "ord-e5f6g7h8",
  "released": true,
  "milestone": "production_started",
  "escrow_status": "production_released",
  "amount_released_usd": 2380.00
}

Stage 7 — MCP Server

OpenFactory exposes all capabilities as an MCP (Model Context Protocol) server, enabling AI agents to discover and call factory tools directly without writing REST integration code.

Connection

# Run the MCP server locally
npx ts-node src/mcp/server.ts

# Claude Desktop config (claude_desktop_config.json)
{
  "mcpServers": {
    "openfactory": {
      "command": "npx",
      "args": ["ts-node", "src/mcp/server.ts"]
    }
  }
}

# For remote MCP over SSE/HTTP
# Connect to:
All 21 tools return structured JSON designed for agent consumption. The MCP server wraps the same REST endpoints documented above, so response schemas are identical.

All 21 MCP Tools

search_factories
Search verified GBA factories by category, MOQ, price tier, and rating. Returns ranked matches with capability data and certifications.
verify_factory_identity
Verify a factory's legal identity — USCC, legal representative, business license expiry, and link to gsxt.gov.cn registry.
get_trust_score
Composite 0-100 trust score with breakdown: identity (USCC + license), execution (on-time milestones), transparency (photo proof), quality (QC pass rate), reputation (reviews).
factory_performance
Earned trust metrics from real transactional data: on-time delivery rate, lead time accuracy, QC pass rate, milestone responsiveness.
get_quote
Request a price quote from a specific factory. Returns unit price, total, lead time, MOQ, and a 7-day valid quote_id for ordering.
get_instant_quote
Sub-second binding quote from a factory's pre-declared pricing rules. No waiting — computed instantly from tiered pricing. Valid 48h.
query_live_capacity
Real-time available capacity across all GBA factories. Returns who can fulfill your order RIGHT NOW with instant pricing. Sub-100ms.
place_order
Place an order from an accepted quote. Payment held in escrow, released after buyer confirms receipt. Returns order_id.
track_order
Check production status — current status, full event history, and estimated ship date.
update_order_status
Update the production milestone of an order (factory-side). Valid statuses: confirmed, in_production, qc, shipped, delivered.
report_milestone
Report a production milestone with enforced ordering. Attach photos and notes for audit trail.
get_milestones
Full milestone timeline for an order — all production milestones, photos, notes, and timestamps.
request_qc_inspection
Request third-party QC inspection (qima, sgs, bureau_veritas, manual). Requires production_started milestone.
get_qc_status
Get QC inspection status for an order — provider, type, status, pass/fail result, and report URL.
lock_deposit
Lock 30% deposit on an order. Transitions escrow from pending_deposit to deposit_held.
check_escrow_status
Current escrow status, full escrow event audit trail, and milestone chain for an order.
confirm_receipt
Buyer confirms receipt. Validates qc_pass prerequisite, then transitions escrow to final_released.
raise_dispute
Raise a dispute on an order. Transitions escrow to disputed state. For quality, delivery, or payment issues.
submit_review
Buyer review with ratings (1-5) for overall, quality, communication, and accuracy. Updates factory trust score.
get_factory_reviews
All buyer reviews for a factory with summary statistics — individual reviews and aggregate averages.
get_analytics
Platform analytics: factory count, quote volume, order count, GMV, and per-factory quote activity.
Typical Agent Flow
search_factories→ verify_factory_identity→ get_instant_quote→ place_order→ lock_deposit→ confirm_receipt
