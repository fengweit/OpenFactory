Escrow
30/40/30 milestone-based payment structure. Deposit locks on order, releases auto-trigger on milestones.
Escrow State Machine pending_deposit ──> deposit_held ──> production_released ──> qc_released ──> final_released (lock 30%) (production_started: 30%) (qc_pass: 40%) (shipped: 30%) | v (dispute) disputed ──> refunded or final_released
POST /orders/:id/escrow/lock JWT Lock 30% deposit ▸

Transitions escrow from pending_deposit to deposit_held. No request body needed — deposit amount (30%) is calculated from the order total.

Example Response
{ "order_id": "ord-001", "deposit_locked": true, "deposit_amount_usd": 6750, "remaining_usd": 15750, "escrow_status": "deposit_held", "message": "Deposit locked. Factory can now begin production." }
POST /orders/:id/escrow/release JWT Release escrow by milestone ▸
Request Body
FieldTypeDescription
milestonestringMilestone triggering release (production_started, qc_pass, shipped)
Auto-Release Triggers
MilestoneReleaseEscrow Transition
production_started30% depositdeposit_held → production_released
qc_pass40% mid-productionproduction_released → qc_released
shipped30% finalqc_released → final_released

QC Inspection
Third-party quality control. Providers: QIMA, SGS, Bureau Veritas, or manual. Auto-creates qc_pass/qc_fail milestones.
POST /orders/:id/qc-request JWT Request QC inspection ▸
Request Body
FieldTypeDescription
providerenumqima | sgs | bureau_veritas | manual
inspection_typeenumduring_production | pre_shipment | container_loading
buyer_idoptstringOptional buyer ID override
Example Response
{ "qc_request_id": "qc-001", "order_id": "ord-001", "provider": "qima", "inspection_type": "pre_shipment", "status": "requested", "created_at": "2026-03-26T10:00:00.000Z" }
POST /orders/:id/qc-result JWT Submit QC result ▸
Request Body
FieldTypeDescription
qc_request_idstringQC request ID
passedbooleantrue = passed, false = failed
report_urloptstringURL to full QC report
notesoptstringInspector notes
Side Effects

Auto-creates a qc_pass or qc_fail milestone on the order. If passed, triggers 40% escrow release (production_released → qc_released).
