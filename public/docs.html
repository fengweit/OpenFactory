API Documentation

Buyer-facing REST API reference with curl examples and response shapes

Authentication

OpenFactory supports two authentication methods: JWT tokens for user sessions and API keys for programmatic / AI-agent access.

JWT Token Flow

For buyers, factories, and admins. Tokens expire after 7 days.

# Register a buyer account
curl -X POST "$BASE_URL/auth/register" \
  -H "Content-Type: application/json" \
  -d '{"email":"buyer@example.com","password":"securepassword","role":"buyer"}'

# Response
{ "user_id": "usr-a1b2c3", "token": "eyJ..." }

# Login
curl -X POST "$BASE_URL/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"email":"buyer@example.com","password":"securepassword"}'

# Response
{ "user_id": "usr-a1b2c3", "role": "buyer", "token": "eyJ..." }

API Key Flow

For AI agents and partner integrations. Keys are prefixed ofk_ and rate-limited per minute.

# Generate key (admin only)
curl -X POST "$BASE_URL/admin/api-keys" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"partner_name":"my-agent","permissions":["GET /factories*","POST /quotes"],"rate_limit_per_min":60}'

# Response
{ "key": "ofk_a1b2c3...", "id": "apk-..." }

# Use in requests
curl -H "Authorization: Bearer ofk_a1b2c3..." "$BASE_URL/factories"
Auth on endpoint cards: Auth = requires Bearer token (JWT or API key). Public = no authentication required.

Base URL

# All endpoints are relative to your deployment origin
BASE_URL="https://your-openfactory-instance.com"

# In browser code, use window.location.origin
# Never hard-code localhost
Rate limits: 100 requests/minute per IP. Auth routes have stricter limits.

GET /factories Auth

Search and filter verified factories. Returns full factory profiles including contact info, certifications, and trust scores.

Query Parameters
NameTypeDescription
categorystringFilter by product category (e.g. pcb_assembly, plastic_injection, cable_assembly)
max_moqnumberMaximum MOQ filter — only return factories with MOQ ≤ this value
verified_onlybooleanSet to true to only return USCC-verified factories
min_trust_scorenumberMinimum trust score (0-100) — computed from delivery, QC, and review data
min_ratingnumberMinimum star rating (1-5)
price_tierstringbudget, mid, or premium
sortstringSort field (e.g. trust_score)
Example
curl -H "Authorization: Bearer $TOKEN" \
  "$BASE_URL/factories?category=pcb_assembly&max_moq=500&verified_only=true&min_trust_score=60"

      "certifications": ["ISO9001", "UL"],
      "verified": true,
      "trust_score": 82,
      "price_tier": "mid",
      "moq": 100,
      "lead_time_days": { "production": 21, "shipping": 7 },

      "rating": 4.5,

POST /quotes Auth

Request a price quote from a specific factory. Returns unit price, total, lead time, MOQ, and a 7-day valid quote_id for ordering.

Request Body
FieldTypeRequiredDescription
factory_idstringYesFactory ID (e.g. "sz-006")
product_descriptionstringYesDescription of the product
quantitynumberYesNumber of units
buyer_idstringNoBuyer identifier
target_price_usdnumberNoTarget unit price in USD
deadline_daysnumberNoDesired lead time in days
Example
curl -X POST "$BASE_URL/quotes" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "factory_id": "sz-006",
    "product_description": "Custom PCB board, 4-layer, 50x30mm",
    "quantity": 5000,
    "target_price_usd": 3.50
  }'
Response
{
  "quote_id": "q-a1b2c3d4",
  "factory_id": "sz-006",
  "unit_price_usd": 2.38,
  "total_price_usd": 11900.00,
  "lead_time_days": 21,
  "moq": 100,
  "valid_until": "2026-04-02T12:00:00.000Z",
  "notes": "Quote for 5000x Custom PCB board, 4-layer, 50x30mm from Nanshan IoT Systems"
}

POST /orders Auth

Place an order from an accepted quote. Payment is held in escrow (30% deposit locked, remainder released on delivery). Returns the order_id for tracking.

Request Body
FieldTypeRequiredDescription
quote_idstringYesThe quote_id from a POST /quotes or instant-quote response
buyer_idstringYesBuyer identifier
Example
curl -X POST "$BASE_URL/orders" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"quote_id": "q-a1b2c3d4", "buyer_id": "buyer-001"}'
Response
{
  "order_id": "ord-e5f6g7h8",
  "quote_id": "q-a1b2c3d4",
  "factory_id": "sz-006",
  "buyer_id": "buyer-001",
  "status": "pending",
  "quantity": 5000,
  "unit_price_usd": 2.38,
  "total_price_usd": 11900.00,
  "escrow_held": true,
  "escrow_status": "pending_deposit",
  "created_at": "2026-03-26T10:30:00.000Z",
  "estimated_ship_date": "2026-04-16T10:30:00.000Z"
}
Order Status Flow
pending→ confirmed→ in_production→ qc→ shipped→ delivered
Escrow State Machine
pending_deposit→ deposit_held→ production_released→ final_released

POST /orders/:id/qc-request Public

Request a third-party QC inspection on an order. Requires the production_started milestone to be reached. Supported providers: qima, sgs, bureau_veritas, manual.

Request Body
FieldTypeRequiredDescription
providerstringYesQC provider: qima, sgs, bureau_veritas, or manual
inspection_typestringNoType of inspection (e.g. "pre_shipment", "during_production")
buyer_idstringNoBuyer identifier
Example
curl -X POST "$BASE_URL/orders/ord-e5f6g7h8/qc-request" \
  -H "Content-Type: application/json" \
  -d '{"provider": "qima", "inspection_type": "pre_shipment"}'
Response 201

  "provider": "qima",

For AI Agents

OpenFactory is designed API-first for AI agent consumption. Every endpoint returns structured JSON with consistent schemas, predictable error shapes, and machine-readable enums — no HTML scraping or natural-language parsing required.

Key design principles for agents:
• All responses are JSON with stable field names — safe to parse programmatically
• Enum values (status, categories, escrow states) are documented and enforced
• Error responses include structured { "error": "..." } messages
• Sub-100ms response times on instant-quote and capacity endpoints
• Rate limit: 100 req/min per API key (request higher limits for production agents)

Typical Agent Workflow

Agent Sourcing Flow
GET /factories→ GET /factories/:id/verify-identity→ GET /factories/:id/instant-quote→ POST /orders→ GET /orders/:id/milestones

API Key Authentication

For automated agents, use API keys (prefixed ofk_) instead of JWT tokens. Request one from an admin via POST /admin/api-keys, then pass it as a Bearer token in the Authorization header.

# All agent requests use the same header
curl -H "Authorization: Bearer ofk_your_api_key" \
  "$BASE_URL/factories?category=pcb_assembly&verified_only=true"

MCP Server

OpenFactory exposes all capabilities as an MCP (Model Context Protocol) server, enabling AI agents to discover and call factory tools directly without writing REST integration code.

Connection

# Run the MCP server locally
npx ts-node src/mcp/server.ts

# Or connect via the MCP client
npx @anthropic/mcp-client --server "npx ts-node src/mcp/server.ts"

# Claude Desktop config (claude_desktop_config.json)
{
  "mcpServers": {
    "openfactory": {
      "command": "npx",
      "args": ["ts-node", "src/mcp/server.ts"]
    }
  }
}

MCP Tools

search_factories
Search verified GBA factories by category, MOQ, price tier, and rating. Returns ranked matches with capability data and certifications.
verify_factory_identity
Verify a factory's legal identity — USCC, legal representative, business license expiry, and link to gsxt.gov.cn registry.
factory_performance
Earned trust metrics from real transactional data: on-time delivery, lead time accuracy, QC pass rate, milestone responsiveness.
query_live_capacity
Real-time available capacity across all GBA factories. Returns who can fulfill your order right now with instant pricing. Sub-100ms.
get_instant_quote
Sub-second binding quote from a factory's pre-declared pricing rules. No waiting — computed instantly from tiered pricing. Valid 48h.
get_quote
Request a price quote from a specific factory. Returns unit price, total, lead time, MOQ, and a 7-day valid quote_id for ordering.
place_order
Place an order from an accepted quote. Payment held in escrow, released after buyer confirms receipt. Returns order_id.
track_order
Check production status — current status, full event history, and estimated ship date.
report_milestone
Report a production milestone with enforced ordering. Attach photos and notes for audit trail.
request_qc_inspection
Request third-party QC inspection (qima, sgs, bureau_veritas, manual). Requires production_started milestone.
lock_deposit
Lock 30% deposit on an order. Transitions escrow from pending_deposit to deposit_held.
confirm_receipt
Buyer confirms receipt. Validates qc_pass prerequisite, then transitions escrow to final_released.
raise_dispute
Raise a dispute on an order. Transitions escrow to disputed state. For quality, delivery, or payment issues.
submit_review
Buyer review with ratings (1-5) for overall, quality, communication, and accuracy. Updates factory trust score.
