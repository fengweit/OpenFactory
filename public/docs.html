OpenFactory API Reference

REST API + MCP tools for AI-native manufacturing sourcing. Version 0.3.0

Quick Start: 5-Call Happy Path

From factory search to payment release in 5 API calls:

1

Search factories

Find verified factories by category, MOQ, and rating.

# Search for electronics factories
curl "$BASE/factories/public?category=electronics_accessories&verified_only=true"
2

Request a quote

Get pricing from a specific factory. Returns a quote_id valid for 7 days.

# Request a quote from factory sz-001
curl -X POST "$BASE/quotes" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"factory_id":"sz-001","product_description":"USB-C cables, 1m braided","quantity":5000}'
3

Place an order

Convert a quote into an order. Escrow is created automatically.

# Place the order
curl -X POST "$BASE/orders" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"quote_id":"q-abc123","buyer_id":"buyer@example.com"}'
4

Report a milestone

Factory reports production progress with photo evidence.

# Factory reports production started
curl -X POST "$BASE/orders/ord-abc123/milestones" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"milestone":"production_started","note":"Assembly line #3 active"}'
5

Release escrow

Milestone-based partial release (30/40/30 structure). Final release after buyer confirms receipt.

# Release escrow for the production_started milestone
curl -X POST "$BASE/orders/ord-abc123/escrow/release" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"milestone":"production_started"}'

Authentication

OpenFactory supports two authentication methods:

JWT Bearer Token

Register an account, then use the token in the Authorization header.

# Register
curl -X POST "$BASE/auth/register" \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"secret123","role":"buyer"}'

# Login (returns { token: "eyJ..." })
curl -X POST "$BASE/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"secret123"}'

# Use the token
curl -H "Authorization: Bearer eyJ..." "$BASE/factories"

API Key

For AI agent integrations. Generated by an admin via POST /admin/api-keys. Pass it via the X-API-Key header.

curl -H "X-API-Key: ofk_..." "$BASE/factories"

Endpoints marked with No Auth are publicly accessible.

Base URL

All examples use $BASE as the server origin. Set it to your deployment URL:

export BASE="https://your-server.com"  # or http://localhost:3000 for local dev

Rate limit: 100 requests/minute per IP. Auth routes are stricter.

Factory Discovery

GET /factories/public No Auth
Search factories without authentication. Returns a curated subset (no contact info). Good for browsing.
ParamTypeDescription
categorystringFilter by product category (e.g. electronics_accessories)
verified_onlybooleanOnly return verified factories
sortstringSort field
min_trust_scorenumberMinimum trust score (0-100)
curl "$BASE/factories/public?category=pcb_assembly&verified_only=true&min_trust_score=60"
GET /factories/:id No Auth
Get full factory details including identity fields (USCC, legal rep, license expiry).
curl "$BASE/factories/sz-001"
GET /factories/:id/verify-identity No Auth
Identity trust data for buyer agents. Returns USCC, legal representative, business license expiry, verification status.
curl "$BASE/factories/sz-001/verify-identity"

# Response:
# { "factory_id": "sz-001", "uscc": "91440300MA5...", "uscc_valid": true,
#   "legal_rep": "Zhang Wei", "business_license_expiry": "2027-03-15",
#   "verified": true, "identity_complete": true }
GET /factories/:id/trust-score No Auth
Composite 0-100 trust score with breakdown: identity, execution, transparency, quality, reputation.
curl "$BASE/factories/sz-001/trust-score"
GET /factories/:id/delivery-performance No Auth
Delivery stats computed from real order data: on-time rate, average lead time, total orders.
curl "$BASE/factories/sz-001/delivery-performance"

Quote → Order Flow

POST /quotes JWT API Key
Create a quote request. Returns pricing, lead time, and a quote_id valid for 7 days.
FieldTypeDescription
factory_idstringrequiredFactory ID (e.g. sz-001)
product_descriptionstringrequiredWhat to manufacture
quantityintegerrequiredNumber of units
buyer_idstringBuyer identifier
target_price_usdnumberTarget unit price in USD
deadline_daysintegerDays until delivery needed
curl -X POST "$BASE/quotes" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "factory_id": "sz-001",
    "product_description": "Custom USB-C cables, 1m, braided nylon",
    "quantity": 5000,
    "target_price_usd": 1.20
  }'

# Response:
# { "quote_id": "q-abc123", "factory_id": "sz-001", "quantity": 5000,
#   "unit_price_usd": 1.35, "total_price_usd": 6750, "lead_time_days": 18 }
POST /orders JWT API Key
Place an order from an accepted quote. Creates a Stripe escrow automatically.
FieldTypeDescription
quote_idstringrequiredQuote ID from POST /quotes
buyer_idstringrequiredBuyer identifier
curl -X POST "$BASE/orders" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"quote_id":"q-abc123","buyer_id":"buyer@example.com"}'
GET /orders/:id JWT API Key
Track an order. Returns order details, milestones, and escrow events.
curl -H "Authorization: Bearer $TOKEN" "$BASE/orders/ord-abc123"

Milestone Tracking

Milestones track production progress with enforced ordering: material_received → production_started → qc_in_progress → qc_pass/qc_fail → ready_for_shipment → shipped.

POST /orders/:id/milestones JWT
Factory reports a production milestone. Only factory/admin roles. May trigger escrow transitions.
FieldTypeDescription
milestonestringrequirede.g. production_started, qc_pass, shipped
photo_urlsstring[]Photo evidence URLs
notestringOptional note (e.g. tracking number)
curl -X POST "$BASE/orders/ord-abc123/milestones" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "milestone": "qc_pass",
    "photo_urls": ["/uploads/milestones/ord-abc123/qc-1.jpg"],
    "note": "All 5000 units passed AQL 2.5 inspection"
  }'
GET /orders/:id/milestones JWT
Get all milestones for an order, enriched with uploaded photos.
curl -H "Authorization: Bearer $TOKEN" \
  "$BASE/orders/ord-abc123/milestones"
POST /orders/:id/milestones/:milestoneId/photos JWT
Upload milestone proof photos (JPEG, PNG, WebP). Max 5 files, 5MB each. Factory/admin only.
curl -X POST "$BASE/orders/ord-abc123/milestones/1/photos" \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@production-line.jpg" \
  -F "file=@qc-report.jpg"

QC Integration

Third-party quality control inspection. Bridges self-reported milestones with independent verification.

POST /orders/:id/qc-request No Auth
Create a QC inspection request. Order must have reached production_started milestone.
FieldTypeDescription
providerstringrequiredqima | sgs | bureau_veritas | manual
inspection_typestringduring_production | pre_shipment | container_loading
buyer_idstringBuyer ID override
curl -X POST "$BASE/orders/ord-abc123/qc-request" \
  -H "Content-Type: application/json" \
  -d '{"provider":"qima","inspection_type":"pre_shipment"}'
POST /orders/:id/qc-result No Auth
Webhook for QC provider to post pass/fail results.
FieldTypeDescription
resultstringrequired"passed" or "failed"
inspector_notesstringInspector's notes
report_urlstringLink to full report
# QC provider callback
curl -X POST "$BASE/orders/ord-abc123/qc-result" \
  -H "Content-Type: application/json" \
  -d '{
    "result": "passed",
    "inspector_notes": "AQL 2.5 -- 0 critical, 1 minor defect",
    "report_url": "https://qima.com/reports/abc123.pdf"
  }'

Escrow Lifecycle

Escrow protects both buyer and factory. Funds flow through a 30/40/30 milestone-gated release structure:

  • 30% released at production_started
  • 40% released at qc_pass
  • 30% released at shipped (final release)

States: pending_deposit → deposit_held → production_released → final_released. Can branch to disputed or refunded.

POST /orders/:id/escrow/lock JWT
Lock 30% deposit. Creates a Stripe PaymentIntent with manual capture. Transitions escrow from pending_deposit to deposit_held.
curl -X POST "$BASE/orders/ord-abc123/escrow/lock" \
  -H "Authorization: Bearer $TOKEN"

# Response:
# { "order_id": "ord-abc123", "deposit_locked": true,
#   "deposit_amount_usd": 2025, "remaining_usd": 4725,
#   "escrow_status": "deposit_held",
#   "payment_intent_id": "pi_...", "escrow_provider": "stripe" }
POST /orders/:id/escrow/release JWT
Milestone-based escrow release with Stripe partial capture. Validates milestone and photo evidence.
FieldTypeDescription
milestonestringrequiredproduction_started | qc_pass | shipped
verified_bystringWho verified (e.g. "qima")
curl -X POST "$BASE/orders/ord-abc123/escrow/release" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"milestone":"qc_pass","verified_by":"qima"}'
GET /orders/:id/escrow-events No Auth
Full escrow audit trail. Every state transition with timestamp, amount, and note.
curl "$BASE/orders/ord-abc123/escrow-events"

# Response:
# { "order_id": "ord-abc123", "escrow_events": [
#   { "from_status": "pending_deposit", "to_status": "deposit_held",
#     "amount_usd": 2025, "note": "30% deposit locked", "created_at": "..." },
#   ... ], "count": 3 }
