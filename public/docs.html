API Documentation

REST API & MCP server reference for OpenFactory

Authentication

OpenFactory supports two authentication methods: JWT tokens for user sessions and API keys for programmatic / AI-agent access.

JWT Token Flow

For buyers, factories, and admins. Tokens expire after 7 days.

// 1. Register
POST /auth/register
{
  "email": "buyer@example.com",
  "password": "securepassword",
  "role": "buyer"           // buyer | factory | admin
}
// Response: { "user_id": "usr-...", "token": "eyJ..." }

// 2. Login
POST /auth/login
{
  "email": "buyer@example.com",
  "password": "securepassword"
}
// Response: { "user_id": "usr-...", "role": "buyer", "token": "eyJ..." }

// 3. Use in requests
Authorization: Bearer eyJ...

API Key Flow

For AI agents and partner integrations. Keys are prefixed ofk_ and rate-limited per minute.

// Generate key (admin only)
POST /admin/api-keys
{
  "partner_name": "my-agent",
  "permissions": ["GET /factories*", "POST /quotes"],
  "rate_limit_per_min": 60
}
// Response: { "key": "ofk_a1b2c3...", "id": "apk-..." }

// Use in requests
Authorization: Bearer ofk_a1b2c3...
Auth requirement labels: JWT JWT only   API Key API key only   JWT | Key Either method   No badge = public endpoint

Base URL

// All endpoints are relative to your deployment origin
GET {origin}/factories/public
GET {origin}/health

In browser code, use window.location.origin to construct URLs. Do not hard-code localhost.

Orders

EndpointAuthDescription
POST /orders JWT | Key Place an order from an accepted quote_id. Payment held in escrow.
GET /orders/:id JWT | Key Full order details with milestones and escrow status
PATCH /orders/:id/status JWT Update order status (factory only): confirmed, in_production, qc, shipped, delivered
GET /orders/:id/health JWT Real-time order health score
Order Status Flow
pending → confirmed → in_production → qc → shipped → delivered

Milestones

Milestones provide granular production tracking beyond order status. Each milestone enforces ordering — you cannot skip steps.

EndpointAuthDescription
POST /orders/:id/milestones JWT Report a milestone. Attach optional photo_urls and note.
GET /orders/:id/milestones JWT Full milestone timeline with photos, notes, timestamps
POST /orders/:id/milestones/:mid/photos JWT Upload milestone photos (multipart, 5MB max per file)
Milestone Chain (enforced order)
material_received → production_started → qc_in_progress → qc_pass → ready_for_shipment → shipped
Branch: qc_in_progress → qc_fail
Prerequisite enforcement: Each milestone requires its predecessors. For example, production_started requires material_received, and ready_for_shipment requires qc_pass. Attempting to skip a step returns 400.

QC Inspections

EndpointDescription
POST /orders/:id/qc-request Request third-party QC. Providers: qima, sgs, bureau_veritas, manual. Types: during_production, pre_shipment, container_loading. Requires production_started milestone.
GET /orders/:id/qc-requests All QC inspection requests with status, pass/fail, and report URL
POST /orders/:id/qc-result QC provider posts result (passed | failed). Auto-creates qc_pass or qc_fail milestone.
External QC blocks self-reporting: If a third-party QC request exists, the factory cannot self-report qc_pass. This makes escrow release credible.

Escrow

EndpointAuthDescription
POST /orders/:id/escrow/lock JWT Lock 30% deposit. Transitions pending_deposit → deposit_held.
POST /orders/:id/escrow/release JWT Milestone-based partial escrow release
GET /orders/:id/escrow-events None Full escrow audit trail with timestamps, triggers, and amounts
POST /orders/:id/release-escrow None Final escrow release (buyer confirms receipt)
Escrow State Machine
pending_deposit → deposit_held → production_released → final_released
Dispute: deposit_held → disputed → refunded | final_released
Auto-transition: production_started milestone triggers deposit_held → production_released
Photo proof required: Escrow release to final_released requires the qc_pass milestone, which in turn requires a QC inspection. If external QC is requested, the factory cannot self-certify. This ensures payment only flows after independently verified quality.

Disputes

EndpointAuthDescription
POST /orders/:id/dispute JWT Raise a dispute. Body: raised_by (buyer | factory), reason, optional evidence_urls. Freezes escrow.
GET /orders/:id/dispute None Get dispute details and resolution status
POST /disputes/:id/resolve JWT Admin resolves: refund_full, refund_partial, or rejected (factory gets paid)

Reviews

EndpointDescription
POST /orders/:id/review Submit buyer review. Ratings (1–5): rating, quality_rating, communication_rating, accuracy_rating, optional comment. One per order. Updates factory trust score.
GET /factories/:id/reviews All reviews for a factory with aggregate averages

MCP Server

OpenFactory exposes all capabilities as an MCP server, enabling AI agents to discover and call factory tools directly.

Connection Info

# Server name & version
name: openfactory
version: 0.2.0
transport: stdio

# Run the MCP server
npx ts-node src/mcp/server.ts

# Claude Desktop config (claude_desktop_config.json)
{
  "mcpServers": {
    "openfactory": {
      "command": "npx",
      "args": ["ts-node", "src/mcp/server.ts"]
    }
  }
}

All 20 Tools

search_factories
Search verified GBA factories by category, MOQ, price tier, and rating. Returns ranked matches with capability data and certifications.
verify_factory_identity
Verify a factory's legal identity — USCC, legal representative, business license expiry, and link to gsxt.gov.cn registry.
factory_performance
Earned trust metrics from real transactional data: on-time delivery, lead time accuracy, QC pass rate, milestone responsiveness.
query_live_capacity
Real-time available capacity across all GBA factories. Returns who can fulfill your order right now with instant pricing. Sub-100ms.
get_instant_quote
Sub-second binding quote from a factory's pre-declared pricing rules. No waiting — computed instantly from tiered pricing. Valid 48h.
get_quote
Request a price quote from a specific factory. Returns unit price, total, lead time, MOQ, and a 7-day valid quote_id for ordering.
place_order
Place an order from an accepted quote. Payment held in escrow, released after buyer confirms receipt. Returns order_id.
track_order
Check production status — current status, full event history, and estimated ship date.
update_order_status
Update production milestone (factory-side). Valid: confirmed, in_production, qc, shipped, delivered. Supports photo attachments.
report_milestone
Report a production milestone with enforced ordering. Attach photos and notes for audit trail.
get_milestones
Full milestone timeline for an order — all milestones, photos, notes, and timestamps.
request_qc_inspection
Request third-party QC inspection (qima, sgs, bureau_veritas, manual). Requires production_started milestone.
get_qc_status
QC inspection status — provider, type, pass/fail result, and report URL.
lock_deposit
Lock 30% deposit on an order. Transitions escrow from pending_deposit to deposit_held.
check_escrow_status
Current escrow status with full audit trail and milestone chain.
confirm_receipt
Buyer confirms receipt. Validates qc_pass prerequisite, then transitions escrow to final_released.
raise_dispute
Raise a dispute on an order. Transitions escrow to disputed state. For quality, delivery, or payment issues.
submit_review
Buyer review with ratings (1–5) for overall, quality, communication, and accuracy. Updates factory trust score.
get_factory_reviews
All buyer reviews for a factory with summary statistics and aggregate averages.
get_analytics
Platform analytics: factory count, quote volume, order count, GMV, and per-factory activity.
