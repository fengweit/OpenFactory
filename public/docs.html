General
Health check, authentication, and OpenAPI spec
GET /health Service health and registered tool list ▶
Returns service status, version, factory counts, uptime, and the list of MCP-registered tools.
Response
{ "service": "OpenFactory API", "version": "0.3.0", "status": "ok", "tools": ["search_factories", "get_quote", "place_order", ...], "total_factories": 10, "verified_factories": 8, "uptime_s": 3600 }
GET /openapi.json OpenAPI 3.0 specification ▶
Returns the full OpenAPI 3.0 spec for machine-readable API discovery.
GET /analytics Platform-wide analytics snapshot ▶
Returns aggregate statistics: total orders, revenue, factory counts, and category breakdown.
Response
{ "total_orders": 42, "total_revenue_usd": 125000, "total_factories": 10, "orders_by_status": { "confirmed": 5, "shipped": 3 } }
POST /auth/register Register a new user account ▶
Create a buyer or factory user. Returns a JWT token for authenticated endpoints.
Request body
{ "email": "buyer@example.com", "password": "securePass123", "role": "buyer", "factory_id": null // required if role=factory }
Response
{ "user_id": "usr-abc123", "email": "buyer@example.com", "role": "buyer", "token": "eyJhbGciOi..." }
POST /auth/login Authenticate and receive JWT ▶
Login with email/password. Returns a JWT bearer token for Authorization header.
Request body
{ "email": "buyer@example.com", "password": "securePass123" }
Response
{ "user_id": "usr-abc123", "token": "eyJhbGciOi..." }

Identity Trust
USCC verification, factory identity, trust scores, and performance metrics
GET /factories/:id/verify-identity Full identity trust data for buyer agents ▶
Returns USCC status, legal rep, license expiry, and verification status. Primary endpoint for AI agent trust evaluation.
Response
{ "factory_id": "sz-001", "verified": true, "uscc": "91440300MA5EXAMPLE8", "uscc_valid": true, "legal_rep": "Zhang Wei", "business_license_expiry": "2028-12-31", "registry_url": "https://gsxt.gov.cn/..." }
POST /factories/:id/verify-uscc Validate and store a USCC code ▶
Submit a Unified Social Credit Code for validation. Verifies the 18-character format and stores it if valid.
Request body
{ "uscc": "91440300MA5EXAMPLE8" }
Response
{ "factory_id": "sz-001", "uscc": "91440300MA5EXAMPLE8", "uscc_valid": true, "verified": true }
GET /factories/:id/trust-score Composite 0-100 trust score ▶
Computes a composite trust score (0-100) combining identity verification, order history, QC pass rate, and dispute rate.
Response
{ "factory_id": "sz-001", "trust_score": 82, "breakdown": { "identity": 25, "transaction_history": 30, "quality": 18, "responsiveness": 9 } }
GET /factories/:id/performance Earned trust metrics from transactional data ▶
Returns performance metrics computed from real orders: on-time delivery rate, QC pass rate, dispute rate, and average lead time.
Response
{ "factory_id": "sz-001", "total_orders": 24, "on_time_rate": 0.92, "qc_pass_rate": 0.96, "dispute_rate": 0.04, "avg_lead_time_days": 18 }

Discovery
Search factories, query live capacity, and manage pricing rules
GET /factories Search and filter all factories ▶
Search factories by category, MOQ, price tier, rating, verification status, and trust score. Primary discovery endpoint for AI agents.
Query parameters
ParamTypeDescription
categorystringFilter by category (e.g. electronics_accessories, pcb_assembly)
max_moqnumberMaximum MOQ filter
price_tierstringbudget | mid | premium
min_ratingnumberMinimum rating (0-5)
verified_onlybooleanOnly return verified factories
sortstringSort field (e.g. trust_score, rating)
min_trust_scorenumberMinimum trust score (0-100)
Response
{ "factories": [ { "id": "sz-001", "name": "Shenzhen Elite Electronics", "verified": true, "trust_score": 82, "identity_complete": true, "categories": ["electronics_accessories"], "moq": 500, "rating": 4.8, "location": { "city": "Shenzhen", "district": "Longhua" } } ], "count": 10 }
GET /factories/:id Single factory with all fields ▶
Returns full factory record including identity fields, capacity, pricing tier, certifications, and trust score.
Response
{ "id": "sz-001", "name": "Shenzhen Elite Electronics", "name_zh": "深圳精英电子", "verified": true, "uscc": "91440300MA5EXAMPLE8", "categories": ["electronics_accessories"], "moq": 500, "rating": 4.8, "trust_score": 82, "capacity_units_per_month": 50000 }
GET /capacity Live capacity query across all factories ▶
Find factories with available capacity for a given category, quantity, and optional lead time constraint.
Query parameters
ParamTypeDescription
categorystring requiredProduct category
qtynumber requiredQuantity needed
max_daysnumberMax lead time in days
Response
{ "query": { "category": "electronics_accessories", "quantity": 1000, "max_days": null }, "count": 3, "results": [ { "factory_id": "sz-001", "available_units": 5000, "unit_price_usd": 2.50, "lead_time_days": 14 } ] }
GET /factories/:id/capacity Get declared capacity for a factory ▶
Returns current declared capacity entries for a single factory, broken down by category.
Response
{ "factory_id": "sz-001", "capacity": [ { "category": "electronics_accessories", "available_units": 5000, "available_from": "2025-02-01" } ] }
PATCH /factories/:id/capacity Update declared capacity ▶
Factory updates its available capacity for a specific category.
Request body
{ "category": "electronics_accessories", "available_units": 8000, "available_from": "2025-03-01", "price_override_usd": 2.30 }
Response
{ "factory_id": "sz-001", "category": "electronics_accessories", "available_units": 8000, "updated": true }
GET /factories/:id/pricing-rules Get all pricing rules for a factory ▶
Returns tiered pricing rules by category and quantity range. Used to compute instant quotes.
Response
{ "factory_id": "sz-001", "rules": [ { "category": "electronics_accessories", "min_qty": 500, "max_qty": 5000, "unit_price_usd": 2.50, "lead_time_days": 14 } ] }
PATCH /factories/:id/pricing-rules Upsert pricing rules ▶
Create or update pricing rules for a factory. Accepts an array of rules.
Request body
[ { "category": "electronics_accessories", "min_qty": 500, "max_qty": 5000, "unit_price_usd": 2.50, "lead_time_days": 14 } ]
Response
{ "factory_id": "sz-001", "rules": [...] }
GET /factories/:id/quotes Quotes received by a factory ▶
Returns all quote requests received by a specific factory.
GET /factories/:id/orders Orders placed with a factory ▶
Returns all orders for a specific factory.

Quoting
Request quotes, broadcast RFQs, get instant quotes from pricing rules
POST /quotes Request a quote from a factory ▶
Submit a quote request to a specific factory. Triggers a WeChat notification to the factory contact.
Request body
{ "factory_id": "sz-001", "product_description": "USB-C charging cable, 1m, braided nylon", "quantity": 5000, "buyer_id": "buyer-123", "target_price_usd": 2.00, "deadline_days": 14 }
Response
{ "quote_id": "q-abc123", "factory_id": "sz-001", "unit_price_usd": 2.50, "total_price_usd": 12500, "lead_time_days": 14, "valid_until": "2025-02-15" }
POST /quotes/:id/respond Factory responds to a quote request ▶
Factory submits their price and lead time for a pending quote request.
Request body
{ "factory_id": "sz-001", "unit_price_usd": 2.30, "lead_time_days": 12, "notes": "Can start production immediately" }
Response
{ "quote_id": "q-abc123", "status": "responded", "unit_price_usd": 2.30, "lead_time_days": 12 }
GET /factories/:id/instant-quote Sub-second binding quote from pricing rules ▶
Returns an instant, binding quote computed from the factory's pricing rules. No human involvement needed.
Query parameters
ParamTypeDescription
categorystring requiredProduct category
qtynumber requiredOrder quantity
Response
{ "factory_id": "sz-001", "category": "electronics_accessories", "quantity": 1000, "unit_price_usd": 2.50, "total_price_usd": 2500, "lead_time_days": 14, "binding": true }
POST /rfq Broadcast RFQ to matching factories ▶
Create an RFQ that auto-matches and notifies qualified factories. Returns the RFQ with matched factory count.
Request body
{ "product_description": "Custom phone case, matte finish, 10 colors", "quantity": 10000, "target_price_usd": 1.50, "categories": ["plastic_injection", "electronics_accessories"], "max_lead_time_days": 21, "buyer_id": "buyer-456" }
Response (201)
{ "rfq_id": "rfq-xyz789", "matched_factories": 4, "quotes_created": 4, "status": "open" }
GET /rfq/:id Get all quote responses for an RFQ ▶
Returns the RFQ details and all factory quote responses, grouped by factory.
Response
{ "rfq_id": "rfq-xyz789", "status": "open", "quotes": [ { "factory_id": "sz-001", "unit_price_usd": 1.40, "lead_time_days": 18 } ] }

Order Execution
Place orders, track status, report milestones, upload proof photos
POST /orders Place an order from an accepted quote Auth ▶
Convert an accepted quote into an order. Triggers WeChat notification, email confirmation, and Stripe escrow creation.
Request body
{ "quote_id": "q-abc123", "buyer_id": "buyer-123" }
Response
{ "order_id": "ord-abc123", "factory_id": "sz-001", "status": "confirmed", "quantity": 5000, "total_price_usd": 12500, "estimated_ship_date": "2025-03-01", "escrow_status": "pending_deposit" }
GET /orders/:id Track order with milestones and escrow ▶
Returns full order details including current status, all milestones, and escrow event history.
Response
{ "order_id": "ord-abc123", "status": "in_production", "factory_id": "sz-001", "total_price_usd": 12500, "milestones": [...], "escrow_events": [...] }
PATCH /orders/:id/status Update order status ▶
Transition order status. Triggers shipping notification email when set to "shipped".
Request body
{ "status": "shipped", "note": "Tracking: SF1234567890", "photo_urls": ["/uploads/proof-001.jpg"] }
Response
{ "order_id": "ord-abc123", "status": "shipped", "updated_at": "2025-02-20T10:30:00Z" }
POST /orders/:id/milestones Factory reports a production milestone Auth ▶
Factory user reports production progress. Certain milestones (e.g. "qc_passed") trigger escrow transitions automatically.
Request body
{ "milestone": "materials_sourced", "photo_urls": ["/uploads/materials-001.jpg"], "note": "All raw materials received and inspected" }
Response (201)
{ "id": 1, "order_id": "ord-abc123", "milestone": "materials_sourced", "reported_by": "factory-user-1", "created_at": "2025-02-10T08:00:00Z", "photo_upload_url": "/orders/ord-abc123/milestones/1/photos" }
GET /orders/:id/milestones All milestones with uploaded photos Auth ▶
Returns all milestones for an order, enriched with uploaded proof photos from the milestone_photos table.
Response
{ "order_id": "ord-abc123", "milestones": [ { "id": 1, "milestone": "materials_sourced", "uploaded_photos": [ { "id": 1, "url": "/uploads/milestones/ord-abc123/1/photo.jpg" } ] } ], "count": 1 }
POST /orders/:id/milestones/:mid/photos Upload milestone proof photos Auth ▶
Upload JPEG, PNG, or WebP images as production proof. Max 5 files, 5MB each. Multipart form data.
Request (multipart/form-data)
// Content-Type: multipart/form-data // Accepted: image/jpeg, image/png, image/webp // Max: 5 files, 5MB each
Response (201)
{ "milestone_id": 1, "order_id": "ord-abc123", "photos": [ { "url": "/uploads/milestones/ord-abc123/1/abc.jpg", "file_size_bytes": 245760 } ], "count": 1 }
GET /orders/:id/health Real-time order health for AI monitoring Auth ▶
Returns health assessment: on-track status, risk flags, milestone completion, and next expected action.
Response
{ "order_id": "ord-abc123", "status": "in_production", "health": "on_track", "days_remaining": 8, "milestones_completed": 3, "milestones_expected": 5, "risk_flags": [] }

Escrow
Lock deposits, release funds, transition escrow states, audit trail
POST /orders/:id/escrow/lock Lock 30% deposit to begin production Auth ▶
Buyer locks a 30% deposit, signaling production can begin. Transitions escrow from pending_deposit to deposit_held.
Response
{ "order_id": "ord-abc123", "deposit_locked": true, "deposit_amount_usd": 3750, "remaining_usd": 8750, "escrow_status": "deposit_held", "message": "Deposit locked. Factory can now begin production." }
POST /orders/:id/release-escrow Buyer confirms receipt, release final escrow ▶
Validates milestone prerequisites (QC passed, shipped), then releases final escrow to the factory via Stripe.
Response
{ "order_id": "ord-abc123", "escrow_event": { "to_status": "final_released", "trigger": "manual" }, "message": "Escrow released. Funds paid to factory within 2 business days." }
POST /orders/:id/escrow-transition Manual escrow state transition (admin) ▶
Admin endpoint to manually transition escrow. State machine: pending_deposit → deposit_held → production_released → final_released.
Request body
{ "to_status": "production_released", "amount_usd": 4375, "note": "Released 35% after QC passed" }
Response
{ "id": 3, "order_id": "ord-abc123", "from_status": "deposit_held", "to_status": "production_released", "amount_usd": 4375, "trigger": "manual", "created_at": "2025-02-18T14:00:00Z" }
GET /orders/:id/escrow-events Full escrow audit trail ▶
Returns complete escrow event history for an order, showing every state transition with timestamps and amounts.
Response
{ "order_id": "ord-abc123", "escrow_events": [ { "from_status": "pending_deposit", "to_status": "deposit_held", "amount_usd": 3750, "created_at": "2025-02-01T10:00:00Z" } ], "count": 1 }
POST /webhooks/stripe Handle Stripe webhook events ▶
Receives Stripe webhook events for payment intents, captures, and refunds. Triggers automated escrow transitions.
Request body (from Stripe)
{ "type": "payment_intent.succeeded", "data": { "object": { "id": "pi_xxx", "amount": 375000 } } }
Response
{ "received": true }

QC (Quality Control)
Request third-party inspections and receive pass/fail results via webhook
POST /orders/:id/qc-request Create a QC inspection request ▶
Buyer creates a quality control inspection request. Specify the QC provider (e.g. SGS, Bureau Veritas, TUV).
Request body
{ "provider": "SGS", "buyer_id": "buyer-123" }
Response (201)
{ "id": 1, "order_id": "ord-abc123", "provider": "SGS", "status": "pending", "created_at": "2025-02-15T09:00:00Z" }
GET /orders/:id/qc-request Get QC inspection status ▶
Retrieve all QC inspection requests and their current status for an order.
Response
{ "order_id": "ord-abc123", "qc_requests": [ { "id": 1, "provider": "SGS", "status": "passed", "inspector_notes": "All units meet spec" } ], "count": 1 }
POST /orders/:id/qc-result QC provider posts pass/fail result (webhook) ▶
Webhook for QC providers to submit inspection results. A "passed" result triggers escrow transition to production_released.
Request body
{ "result": "passed", "inspector_notes": "All 5000 units inspected. 99.8% pass rate.", "report_url": "https://sgs.com/reports/12345.pdf" }
Response
{ "order_id": "ord-abc123", "qc_result": "passed", "escrow_transition": { "to_status": "production_released" } }

Disputes
Raise disputes, submit evidence, and resolve with refund or rejection
POST /orders/:id/dispute Raise a dispute on an order Auth ▶
Buyer or factory raises a dispute. Freezes escrow and creates a dispute record. Evidence URLs can be attached.
Request body
{ "raised_by": "buyer", "reason": "Product does not match sample. Color is different.", "evidence_urls": ["/uploads/evidence-001.jpg"] }
Response (201)
{ "id": 1, "order_id": "ord-abc123", "raised_by": "buyer", "reason": "Product does not match sample.", "status": "open", "created_at": "2025-02-20T16:00:00Z" }
GET /orders/:id/dispute Get disputes for an order ▶
Returns all disputes for an order including status, evidence, and resolution details.
Response
{ "order_id": "ord-abc123", "disputes": [ { "id": 1, "raised_by": "buyer", "status": "open" } ], "count": 1 }
POST /disputes/:id/resolve Platform resolves a dispute (admin) Auth ▶
Admin resolves with refund_full, refund_partial, or rejected. Updates escrow accordingly.
Request body
{ "resolution": "refund_partial", "resolution_notes": "50% refund agreed. Color variance acceptable but not matching sample." }
Response
{ "id": 1, "status": "resolved", "resolution": "refund_partial", "resolved_at": "2025-02-22T10:00:00Z" }

Onboarding
Factory applications, admin approval, and quick-reply magic links
POST /onboard Submit factory application ▶
Submit a new factory application with USCC, categories, certifications, and contact info. USCC is required and validated.
Request body
{ "name_en": "Shenzhen Quality Electronics", "name_zh": "深圳品质电子", "city": "Shenzhen", "categories": ["electronics_accessories"], "certifications": ["ISO9001", "CE"], "moq": 500, "uscc": "91440300MA5EXAMPLE8", "contact_name": "Wang Wei", "wechat_id": "wangwei_factory" }
Response
{ "application_id": "app-xyz789", "status": "pending", "message": "Application received. Our team will contact you within 2 business days." }
GET /admin/applications List factory applications (admin) ▶
List all factory applications. Optional status filter: pending, approved, rejected.
Query parameters
ParamTypeDescription
statusstringFilter: pending, approved, rejected
Response
{ "applications": [...], "count": 5 }
POST /admin/applications/:id/approve Approve and create factory record ▶
Approves a pending application and creates the factory record with identity fields.
Response
{ "approved": true, "factory_id": "sz-011", "message": "Application approved. Factory record created." }
GET /factory/quick-reply Magic link redirect from WeChat ▶
Redirects factory owners from WeChat notification links to the mobile portal with pre-filled parameters.
Query parameters
ParamTypeDescription
fstringFactory ID
astringAction: quotes, orders
tstringToken (optional)
POST /test/notify Send a test WeChat notification ▶
Sends a test WeChat notification to a factory. No auth required. Useful for verifying webhook integration.
Query parameters
ParamTypeDescription
factory_idstringFactory ID (default: sz-001)
Response
{ "sent": true, "webhook_url": "https://qyapi.weixin.qq.com/...", "payload": { ... } }
