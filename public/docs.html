API Reference

Complete REST API documentation for OpenFactory, grouped by trust workflow layer. All endpoints return JSON. Rate limit: 100 req/min per IP.

Try It

Base URL is auto-detected. Paste a JWT token or API key above, then copy curl examples below.

Identity Trust
Verify factory legal identity via USCC, legal representative, and business license.
GET /factories/:id/verify-identity Public ▶

Returns a factory's legal identity data: USCC (Unified Social Credit Code), legal representative, business license expiry, verification status, and a link to the Chinese national business registry.

Path Parameters
NameTypeDescription
idstringFactory ID (e.g. sz-001)
Example Response
{ "factory_id": "sz-001", "name": "Shenzhen Precision Electronics", "uscc": "91440300MA5EXAMPLE1", "legal_rep": "Zhang Wei", "business_license_expiry": "2028-12-31", "verified": true, "registry_url": "https://gsxt.gov.cn" }
curl
# Verify factory identity curl "${BASE}/factories/sz-001/verify-identity"
POST /factories/:id/verify-uscc Public ▶

Validate and store a USCC (Unified Social Credit Code) for a factory. The USCC must be exactly 18 alphanumeric characters matching the Chinese standard format.

Path Parameters
NameTypeDescription
idstringFactory ID
Request Body
FieldTypeDescription
usccstring18-character USCC code (regex: /^[0-9A-Z]{18}$/)
curl
# Validate USCC curl -X POST "${BASE}/factories/sz-001/verify-uscc" \ -H "Content-Type: application/json" \ -d '{"uscc": "91440300MA5EXAMPLE1"}'
GET /factories/:id/trust-score Public ▶

Returns a composite 0-100 trust score computed from identity verification, transaction history, delivery performance, QC pass rate, and buyer reviews.

Path Parameters
NameTypeDescription
idstringFactory ID
Example Response
{ "factory_id": "sz-001", "trust_score": 78, "breakdown": { "identity": 25, "transaction_history": 20, "delivery_performance": 18, "qc_pass_rate": 10, "buyer_reviews": 5 } }
curl
curl "${BASE}/factories/sz-001/trust-score"

Capability Trust
Search factories, query live capacity, and get instant binding quotes.
GET /factories JWT / API Key ▶

Search verified GBA factories by category, MOQ, price tier, rating, and trust score. Returns full factory records with identity fields. Requires authentication (JWT or API key).

Query Parameters
NameTypeDescription
categorystringFilter by category (e.g. pcb_assembly, electronics_accessories, plastic_injection, metal_enclosure, cable_assembly)
max_moqnumberMaximum acceptable MOQ
price_tierstringbudget | mid | premium
min_ratingnumberMinimum rating 0-5
verified_onlybooleanOnly verified factories
sortstringSort field (e.g. trust_score)
min_trust_scorenumberMinimum trust score 0-100
Response Schema (per factory, from FactoryCapabilitySchema)
FieldTypeDescription
idstringFactory ID
namestringEnglish name
name_zhstringChinese name
location{ city, district }Factory location
categoriesstring[]Product categories
moqnumberMinimum order quantity
lead_time_days{ sample, production }Lead times in days
certificationsstring[]e.g. ISO9001, CE, FCC
price_tierstringbudget | mid | premium
capacity_units_per_monthnumberMonthly production capacity
verifiedbooleanOn-site verified
ratingnumber0-5 average buyer rating
trust_scorenumber|nullComposite trust score 0-100
usccstringUnified Social Credit Code (18 chars)
legal_repstringLegal representative name
curl
# Search factories (with API key) curl "${BASE}/factories?category=pcb_assembly&verified_only=true" \ -H "x-api-key: YOUR_API_KEY" # Search factories (with JWT) curl "${BASE}/factories?min_trust_score=50&sort=trust_score" \ -H "Authorization: Bearer YOUR_JWT"

Order Execution Trust
Place orders, report milestones with photo evidence, and monitor order health.
POST /orders JWT / API Key ▶

Place a manufacturing order from an accepted quote. Payment is held in escrow and only released after buyer confirms receipt. Triggers WeChat notification to factory, Stripe escrow creation, and order confirmation email.

Request Body
FieldTypeDescription
quote_idstringQuote ID from /quotes (valid 7 days)
buyer_idstringBuyer identifier
Response Schema (OrderSchema)
FieldTypeDescription
order_idstringGenerated order ID
quote_idstringSource quote
factory_idstringFactory fulfilling the order
buyer_idstringBuyer who placed the order
statusstringpending | confirmed | in_production | qc | shipped | delivered | disputed
quantitynumberUnits ordered
unit_price_usdnumberPrice per unit
total_price_usdnumberTotal order value
escrow_heldbooleanWhether escrow is active
escrow_statusstringpending_deposit | deposit_held | production_released | final_released | disputed | refunded
estimated_ship_datestringISO date
curl
curl -X POST "${BASE}/orders" \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_JWT" \ -d '{"quote_id": "q-abc123", "buyer_id": "buyer-001"}'
POST /orders/:id/milestones JWT (factory/admin) ▶

Factory reports a production milestone. Enforces milestone ordering (e.g. production_started requires material_received). Certain milestones automatically trigger escrow state transitions. Only factory users and admins can report milestones.

Request Body
FieldTypeDescription
milestonestringmaterial_received | production_started | qc_in_progress | qc_pass | qc_fail | ready_for_shipment | shipped
photo_urlsstring[]Photo URLs documenting this milestone
notestringNote (e.g. QC results, tracking number)
curl
curl -X POST "${BASE}/orders/ord-abc123/milestones" \ -H "Content-Type: application/json" \ -H "Authorization: Bearer FACTORY_JWT" \ -d '{ "milestone": "production_started", "photo_urls": ["https://example.com/photo1.jpg"], "note": "Production line A running" }'
POST /orders/:id/milestones/:mid/photos JWT (factory/admin) ▶

Upload milestone proof photos as multipart form data. Accepts JPEG, PNG, and WebP images up to 5MB each, max 5 files per request. Photos are stored and linked to the milestone for audit trail.

Path Parameters
NameTypeDescription
idstringOrder ID
midstringMilestone ID
Request Body
FieldTypeDescription
filesmultipartImage files (JPEG, PNG, WebP). Max 5MB each, max 5 files.
curl
curl -X POST "${BASE}/orders/ord-abc123/milestones/1/photos" \ -H "Authorization: Bearer FACTORY_JWT" \ -F "file=@production_photo.jpg" \ -F "file=@qc_closeup.png"
GET /orders/:id/health JWT ▶

Real-time order health for AI agent monitoring. Returns current status, milestone progress, schedule adherence, and risk indicators. Designed for autonomous agent workflows that need to decide when to intervene.

Path Parameters
NameTypeDescription
idstringOrder ID
curl
curl "${BASE}/orders/ord-abc123/health" \ -H "Authorization: Bearer YOUR_JWT"

Payment Trust
Milestone-based escrow: lock deposits, release on verified milestones, full audit trail.
POST /orders/:id/escrow/lock JWT ▶

Lock a 30% deposit on an order. Transitions escrow from pending_deposit to deposit_held. Call right after placing an order to commit buyer intent. Factory can begin production once deposit is locked.

Path Parameters
NameTypeDescription
idstringOrder ID
Example Response
{ "order_id": "ord-abc123", "deposit_locked": true, "deposit_amount_usd": 6300, "remaining_usd": 14700, "escrow_status": "deposit_held", "message": "Deposit locked. Factory can now begin production." }
curl
curl -X POST "${BASE}/orders/ord-abc123/escrow/lock" \ -H "Authorization: Bearer YOUR_JWT"
POST /orders/:id/escrow/release JWT ▶

Milestone-based escrow release. Validates that the required milestone has been reached and photo evidence exists before transitioning escrow. Prevents premature payment release.

Request Body
FieldTypeDescription
milestonestringproduction_started | qc_pass | shipped
verified_bystringWho verified (e.g. qima, platform)
curl
curl -X POST "${BASE}/orders/ord-abc123/escrow/release" \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_JWT" \ -d '{"milestone": "qc_pass", "verified_by": "qima"}'
GET /orders/:id/escrow-events Public ▶

Full escrow audit trail for an order. Returns every escrow state transition with timestamp, trigger, amount, and notes. Use for transparency and dispute evidence.

Example Response
{ "order_id": "ord-abc123", "escrow_events": [ { "from_status": "pending_deposit", "to_status": "deposit_held", "triggered_by": "manual", "amount_usd": 6300, "timestamp": "2026-03-26T10:00:00Z" } ], "count": 1 }
curl
curl "${BASE}/orders/ord-abc123/escrow-events"

QC / Dispute
Third-party quality control inspections and dispute resolution.
POST /orders/:id/qc-request Public ▶

Request a third-party QC inspection. The order must have at least the production_started milestone. This bridges self-reported milestones and independently verified quality, making escrow release credible. Supported providers: QIMA, SGS, Bureau Veritas, or manual inspection.

Request Body
FieldTypeDescription
providerstringqima | sgs | bureau_veritas | manual
inspection_typestringduring_production | pre_shipment | container_loading
buyer_idstringBuyer ID override
curl
curl -X POST "${BASE}/orders/ord-abc123/qc-request" \ -H "Content-Type: application/json" \ -d '{ "provider": "qima", "inspection_type": "pre_shipment" }'
POST /orders/:id/dispute JWT ▶

Raise a dispute on an order. Transitions escrow to disputed state, freezing funds until resolution. Can be raised by buyer, factory, or platform.

Request Body
FieldTypeDescription
raised_bystringbuyer | factory | platform
reasonstringReason for the dispute
evidence_urlsstring[]Photo/document evidence URLs
curl
curl -X POST "${BASE}/orders/ord-abc123/dispute" \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_JWT" \ -d '{ "raised_by": "buyer", "reason": "Product does not match approved sample", "evidence_urls": ["https://example.com/evidence1.jpg"] }'
