API Reference

Complete REST API documentation for OpenFactory. Every endpoint your AI agent needs to source from verified Chinese manufacturers.

Base URL:
🔑
Authentication
Endpoints marked AUTH require a Bearer token from /auth/login. Rate limit: 100 req/min per IP.
# Include in request headers for authenticated endpoints Authorization: Bearer <your_jwt_token>
🔍
Discovery
Search factories, query live capacity, and pull platform analytics.
GET /factories Search & filter factories ▾
Query Parameters
ParamTypeDescription
categorystring optionalelectronics_accessories | pcb_assembly | plastic_injection | metal_enclosure | cable_assembly
max_moqnumber optionalMaximum MOQ threshold
price_tierstring optionalbudget | mid | premium
min_ratingnumber optionalMinimum rating (0-5)
verified_onlystring optionaltrue to show only verified factories
Example Request
curl "$BASE/factories?category=pcb_assembly&verified_only=true"
Example Response
{ "factories": [ { "id": "sz-006", "name": "Nanshan IoT Systems", "name_zh": "南山物联网系统", "city": "Shenzhen", "categories": ["pcb_assembly"], "verified": true, "rating": 4.6, "moq": 200, "identity_complete": true } ], "count": 1 }
GET /factories/:id Single factory with all fields ▾
Path Parameters
ParamTypeDescription
idstring requiredFactory ID, e.g. sz-006
Example Request
curl "$BASE/factories/sz-006"
Example Response
{ "id": "sz-006", "name": "Nanshan IoT Systems", "name_zh": "南山物联网系统", "city": "Shenzhen", "district": "Nanshan", "categories": "pcb_assembly", "moq": 200, "verified": 1, "rating": 4.6, "uscc": "91440300MA5F4X2T3N", "legal_rep": "Chen Wei", "business_license_expiry": "2028-12-31" }

Identity
Factory identity verification and trust metrics. USCC validation, legal rep, and earned performance scores.
GET /factories/:id/verify-identity Identity trust data for buyer agents ▾
Path Parameters
ParamTypeDescription
idstring requiredFactory ID
Example Request
curl "$BASE/factories/sz-006/verify-identity"
Example Response
{ "factory_id": "sz-006", "verified": true, "uscc": "91440300MA5F4X2T3N", "uscc_valid": true, "legal_rep": "Chen Wei", "business_license_expiry": "2028-12-31", "identity_complete": true }
POST /factories/:id/verify-uscc Validate & store USCC ▾
Path Parameters
ParamTypeDescription
idstring requiredFactory ID
Request Body
FieldTypeDescription
usccstring required18-character alphanumeric USCC (regex: /^[0-9A-Z]{18}$/)
Example Request
curl -X POST "$BASE/factories/sz-006/verify-uscc" \ -H "Content-Type: application/json" \ -d '{"uscc": "91440300MA5F4X2T3N"}'
Example Response
{ "factory_id": "sz-006", "uscc": "91440300MA5F4X2T3N", "uscc_valid": true, "verified": true }
GET /factories/:id/performance Earned trust metrics ▾

Computed from transactional data: on-time delivery, QC pass rates, dispute frequency.

Path Parameters
ParamTypeDescription
idstring requiredFactory ID
Example Request
curl "$BASE/factories/sz-006/performance"
Example Response
{ "factory_id": "sz-006", "total_orders": 42, "on_time_rate": 0.95, "qc_pass_rate": 0.98, "dispute_rate": 0.02, "avg_lead_days": 18 }

Quoting
Instant quotes from pricing rules, full quote requests, and factory responses.
GET /factories/:id/instant-quote Sub-second binding quote ▾
Query Parameters
ParamTypeDescription
categorystring requiredProduct category
qtynumber requiredOrder quantity (> 0)
Example Request
curl "$BASE/factories/sz-006/instant-quote?category=pcb_assembly&qty=5000"
Example Response
{ "factory_id": "sz-006", "factory_name": "Nanshan IoT Systems", "category": "pcb_assembly", "quantity": 5000, "unit_price_usd": 4.20, "total_price_usd": 21000, "lead_time_days": 21 }
POST /quotes Request a quote from factory ▾
Request Body
FieldTypeDescription
factory_idstring requiredTarget factory ID
product_descriptionstring requiredWhat you want manufactured
quantitynumber requiredOrder quantity
buyer_idstring optionalBuyer identifier
target_price_usdnumber optionalTarget unit price in USD
deadline_daysnumber optionalDesired lead time in days
Example Request
curl -X POST "$BASE/quotes" \ -H "Content-Type: application/json" \ -d '{ "factory_id": "sz-006", "product_description": "Custom PCB for IoT sensor module", "quantity": 5000, "target_price_usd": 4.00 }'
Example Response
{ "quote_id": "q-abc123", "factory_id": "sz-006", "unit_price_usd": 4.20, "total_price_usd": 21000, "lead_time_days": 21, "valid_until": "2026-04-10T00:00:00Z" }
POST /quotes/:id/respond Factory responds to quote ▾
Request Body
FieldTypeDescription
factory_idstring requiredFactory responding
unit_price_usdnumber requiredOffered unit price
lead_time_daysnumber requiredOffered lead time
notesstring optionalAdditional notes
Example Request
curl -X POST "$BASE/quotes/q-abc123/respond" \ -H "Content-Type: application/json" \ -d '{"factory_id":"sz-006","unit_price_usd":4.10,"lead_time_days":18}'
Example Response
{ "quote_id": "q-abc123", "status": "responded", "unit_price_usd": 4.10, "lead_time_days": 18 }
GET /factories/:id/pricing-rules Get pricing rules ▾

Request Body
FieldTypeDescription

Execution

POST /orders/:id/milestones Report production milestone AUTH ▾

Only factory or admin users can report milestones. Certain milestones trigger automatic escrow transitions.

Request Body
FieldTypeDescription
milestonestring requiredmaterials_sourced | production_started | production_50 | production_complete | qc_passed | shipped
photo_urlsstring[] optionalEvidence photo URLs
notestring optionalMilestone notes
Example Request
curl -X POST "$BASE/orders/ord-xyz789/milestones" \ -H "Content-Type: application/json" \ -H "Authorization: Bearer <token>" \ -d '{"milestone":"production_started","note":"Assembly line 3 allocated"}'
Example Response — 201 Created
{ "id": 1, "order_id": "ord-xyz789", "milestone": "production_started", "reported_by": "factory-user-001", "timestamp": "2026-03-26T10:00:00Z", "note": "Assembly line 3 allocated" }

Payment
Escrow lifecycle: lock deposit, production, release on delivery.
POST /orders/:id/escrow/lock Lock 30% deposit AUTH ▾

Locks 30% of order total as deposit. Called right after order creation to commit buyer intent. Escrow must be in pending_deposit state.

Path Parameters
ParamTypeDescription
idstring requiredOrder ID
Example Request
curl -X POST "$BASE/orders/ord-xyz789/escrow/lock" \ -H "Authorization: Bearer <token>"
Example Response
{ "order_id": "ord-xyz789", "deposit_locked": true, "deposit_amount_usd": 6300, "remaining_usd": 14700, "escrow_status": "deposit_held", "message": "Deposit locked. Factory can now begin production." }
POST /orders/:id/release-escrow Release final escrow ▾

Buyer confirms receipt, releasing remaining escrow to factory. Validates milestone prerequisites (QC passed, shipped) before allowing release.

Example Request
curl -X POST "$BASE/orders/ord-xyz789/release-escrow"
Example Response
{ "order_id": "ord-xyz789", "status": "released", "escrow_event": { "to_status": "final_released" }, "message": "Escrow released. Funds will be paid to factory within 2 business days." }
GET /orders/:id/escrow-events Full escrow audit trail ▾
Example Request
curl "$BASE/orders/ord-xyz789/escrow-events"
Example Response
{ "order_id": "ord-xyz789", "escrow_events": [ { "to_status": "deposit_held", "trigger": "manual", "amount_usd": 6300, "timestamp": "2026-03-26T09:00:00Z" }, { "to_status": "final_released", "trigger": "manual", "timestamp": "2026-04-16T14:00:00Z" } ], "count": 2 }
POST /orders/:id/escrow-transition Manual escrow transition (admin) ▾
Request Body
FieldTypeDescription
to_statusstring requiredpending_deposit | deposit_held | production_released | final_released | refunded
amount_usdnumber optionalAmount for this transition
notestring optionalAdmin note
Example Request
curl -X POST "$BASE/orders/ord-xyz789/escrow-transition" \ -H "Content-Type: application/json" \ -d '{"to_status":"production_released","note":"50% milestone reached"}'

Disputes
Raise and resolve disputes. Escrow is frozen during active disputes.
POST /orders/:id/dispute Raise a dispute AUTH ▾
Request Body
FieldTypeDescription
raised_bystring requiredbuyer | factory | platform
reasonstring requiredDescription of the dispute
evidence_urlsstring[] optionalEvidence photo/document URLs
Example Request
curl -X POST "$BASE/orders/ord-xyz789/dispute" \ -H "Content-Type: application/json" \ -H "Authorization: Bearer <token>" \ -d '{ "raised_by": "buyer", "reason": "Units received with visible solder defects on 12% of boards", "evidence_urls": ["https://photos.example.com/defect-1.jpg"] }'
Example Response — 201 Created
{ "id": 1, "order_id": "ord-xyz789", "raised_by": "buyer", "reason": "Units received with visible solder defects on 12% of boards", "status": "open", "created_at": "2026-03-26T14:00:00Z" }
GET /orders/:id/dispute Get disputes for order ▾
Example Request
curl "$BASE/orders/ord-xyz789/dispute"
Example Response
{ "order_id": "ord-xyz789", "disputes": [ { "id": 1, "raised_by": "buyer", "status": "open", "reason": "Solder defects" } ], "count": 1 }
POST /disputes/:id/resolve Resolve dispute (admin) AUTH ▾
Request Body
FieldTypeDescription
resolutionstring requiredrefund_full | refund_partial | rejected
resolution_notesstring optionalExplanation of resolution decision
Example Request
curl -X POST "$BASE/disputes/1/resolve" \ -H "Content-Type: application/json" \ -H "Authorization: Bearer <token>" \ -d '{"resolution":"refund_partial","resolution_notes":"12% refund for defective units"}'
Example Response
{ "id": 1, "status": "resolved", "resolution": "refund_partial", "resolution_notes": "12% refund for defective units", "resolved_at": "2026-03-27T10:00:00Z" }

Request Body
FieldTypeDescription

rolestring optionalbuyer | factory

Example Response

POST /auth/login Login & get JWT ▾
Request Body
FieldTypeDescription
emailstring requiredUser email

Example Request
curl -X POST "$BASE/auth/login" \ -H "Content-Type: application/json" \ -d '{"email":"buyer@example.com","password":"securepass123"}'

{ "token": "eyJhbGciOiJIUzI1NiIs...", "user_id": "usr-abc123", "role": "buyer" }

FieldTypeDescription

curl -X POST "$BASE/onboard" \ -H "Content-Type: application/json" \ -d '{ "name_en": "Shenzhen Precision Electronics", "name_zh": "深圳精密电子", "categories": ["pcb_assembly"], "moq": 500, "wechat_id": "sz_precision" }'

{ "application_id": "app-def456", "status": "pending", "message": "Application received. Our Shenzhen team will contact you within 2 business days." }
