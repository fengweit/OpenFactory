API Reference

25+ REST endpoints + 21 MCP tools. Search verified Chinese manufacturers, get instant quotes, place escrow-protected orders, and track production milestones with photo evidence.

Two interfaces, one platform: Every capability is available as both a REST API (for HTTP clients) and an MCP tool (for AI agents). Sections below show both.

Authentication

OpenFactory supports JWT tokens for user sessions and API keys (ofk_ prefix) for AI agent / partner access. Most read endpoints are public; write operations require auth.

Register & Login

# Register
curl -X POST "$BASE_URL/auth/register" \
  -H "Content-Type: application/json" \
  -d '{"email":"buyer@example.com","password":"secret123","role":"buyer"}'

# Login
curl -X POST "$BASE_URL/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"email":"buyer@example.com","password":"secret123"}'

# Response (both)
{ "user_id": "usr-a1b2c3", "role": "buyer", "token": "eyJhbG..." }
FieldTypeRequiredDescription
emailstringYesUser email
passwordstringYesPassword
rolestringNobuyer (default) | factory
factory_idstringNoRequired when role is factory

API Keys (for AI agents)

Keys are prefixed ofk_ and support per-key rate limits + endpoint permissions.

# Generate API key (admin only)
curl -X POST "$BASE_URL/admin/api-keys" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"partner_name":"my-sourcing-agent","permissions":["GET /factories*","POST /quotes","POST /orders"],"rate_limit_per_min":60}'

# Use in any request
curl -H "Authorization: Bearer ofk_a1b2c3..." "$BASE_URL/factories"

MCP Connection (no auth needed)

AI agents using MCP connect via stdio — no tokens needed.

// claude_desktop_config.json
{
  "mcpServers": {
    "openfactory": {
      "command": "npx",
      "args": ["ts-node", "src/mcp/server.ts"]
    }
  }
}
Auth badges on endpoints: Auth = Bearer token (JWT or API key). JWT or Key = either works. Public = no auth. tool = MCP equivalent.

Base URL & Errors

# Set your base URL (never hard-code localhost in client code)
BASE_URL="https://your-openfactory-instance.com"

# Health check
curl "$BASE_URL/health"

# OpenAPI 3.0.3 spec
curl "$BASE_URL/openapi.json"

Rate limit: 100 req/min per IP. All errors return JSON:

StatusMeaning
400Bad request — invalid input or missing required fields
401Unauthorized — missing or invalid Bearer token
403Forbidden — insufficient role permissions
404Not found
409Conflict — invalid state transition (e.g. escrow already locked)
422Unprocessable — invalid status transition
429Rate limited

Report Milestone

Milestones are the backbone of the trust system. Factories report production progress with photo evidence, triggering escrow releases, trust score updates, and buyer notifications.

POST /orders/:id/milestones Auth (factory/admin) report_milestone

REST
POST /orders/:id/milestones
MCP
report_milestone({ order_id, milestone, photo_urls, note })
FieldTypeRequiredDescription
milestonestringYesOne of the 7 milestone types (see below)
photo_urlsstring[]NoPhoto evidence URLs
notestringNoProduction note

The 7 Milestone Types

material_received
Raw materials arrived at factory
production_started
Manufacturing has begun
Triggers escrow release
production_50
Production 50% complete
production_complete
All units manufactured
qc_pass
Quality control passed
Triggers escrow release
packed
Order packed and ready to ship
shipped
Order dispatched to buyer
Triggers escrow release
# Report production started with photo proof
curl -X POST "$BASE_URL/orders/ord-e5f6g7h8/milestones" \
  -H "Authorization: Bearer $FACTORY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "milestone": "production_started",
    "photo_urls": ["/uploads/milestones/ord-e5f6g7h8/line-running.jpg"],
    "note": "Production line A running, 500 units/day target"
  }'

# Response 201 — includes escrow transition if triggered
{
  "id": 1,
  "order_id": "ord-e5f6g7h8",
  "milestone": "production_started",
  "reported_by": "usr-factory1",
  "created_at": "2026-03-28T08:00:00Z",
  "photo_upload_url": "/orders/ord-e5f6g7h8/milestones/1/photos",
  "escrow_transition": {
    "from": "deposit_held",
    "to": "production_partial",
    "amount_usd": 6300
  }
}

Escrow Flow

Milestone-gated escrow: funds only release when real production progress is verified with photo evidence.

Escrow State Machine
pending_deposit→ deposit_held (30%)→ production_partial→ qc_released→ shipped_released→ final_released
any state→ disputed→ resolved
TransitionTriggerAmount
pending_deposit → deposit_heldBuyer locks deposit30% of total
deposit_held → production_partialproduction_started milestoneAuto-release on milestone
production_partial → qc_releasedqc_pass milestoneAuto-release on milestone
qc_released → shipped_releasedshipped milestoneAuto-release on milestone
shipped_released → final_releasedBuyer confirms receiptRemaining balance
any → disputedDispute raisedFunds frozen

Complete Trust Workflow

The end-to-end flow: place order → report milestone with photo → trigger QC → escrow auto-releases. Copy these curl commands and run in sequence.

Before starting: Set BASE_URL to your deployment URL. Get a buyer token from /auth/login and a factory token for milestone reporting. 
##############################################
# STEP 1: Get quote and place order
##############################################

# 1a. Get instant quote
curl "$BASE_URL/factories/sz-006/instant-quote?category=pcb_assembly&qty=5000"

# 1b. Request formal quote (if you need custom specs)
curl -X POST "$BASE_URL/quotes" \
  -H "Authorization: Bearer $BUYER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "factory_id": "sz-006",
    "product_description": "Custom PCB assembly, 4-layer, lead-free",
    "quantity": 5000
  }'

# 1c. Place order (use quote_id from step 1b)
curl -X POST "$BASE_URL/orders" \
  -H "Authorization: Bearer $BUYER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"quote_id": "QUOTE_ID", "buyer_id": "buyer@example.com"}'


##############################################
# STEP 2: Lock 30% escrow deposit
##############################################

curl -X POST "$BASE_URL/orders/ORDER_ID/escrow/lock" \
  -H "Authorization: Bearer $BUYER_TOKEN"


##############################################
# STEP 3: Factory reports milestones with photos
##############################################

# 3a. Report production started (triggers escrow release)
curl -X POST "$BASE_URL/orders/ORDER_ID/milestones" \
  -H "Authorization: Bearer $FACTORY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "milestone": "production_started",
    "note": "Line A running, 500 units/day"
  }'

# 3b. Upload photo proof
curl -X POST "$BASE_URL/orders/ORDER_ID/milestones/MILESTONE_ID/photos" \
  -H "Authorization: Bearer $FACTORY_TOKEN" \
  -F "photo=@production-line.jpg"


##############################################
# STEP 4: Trigger third-party QC inspection
##############################################

# 4a. Buyer requests QC
curl -X POST "$BASE_URL/orders/ORDER_ID/qc-request" \
  -H "Content-Type: application/json" \
  -d '{"provider": "qima", "inspection_type": "pre_shipment"}'

# 4b. QC provider posts result via webhook
curl -X POST "$BASE_URL/orders/ORDER_ID/qc-result" \
  -H "Content-Type: application/json" \
  -d '{
    "result": "passed",
    "inspector_notes": "AQL 2.5 — 0 critical, 2 minor defects",
    "report_url": "https://qima.com/reports/INS-2026-12345"
  }'

# 4c. Factory reports QC pass (triggers escrow release)
curl -X POST "$BASE_URL/orders/ORDER_ID/milestones" \
  -H "Authorization: Bearer $FACTORY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "milestone": "qc_pass",
    "photo_urls": ["/uploads/qc-report.jpg"],
    "note": "QC passed — Qima report INS-2026-12345"
  }'


##############################################
# STEP 5: Ship and release final escrow
##############################################

# 5a. Factory reports shipped (triggers escrow release)
curl -X POST "$BASE_URL/orders/ORDER_ID/milestones" \
  -H "Authorization: Bearer $FACTORY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "milestone": "shipped",
    "note": "DHL tracking: 1234567890"
  }'

# 5b. Buyer confirms receipt (releases remaining escrow)
curl -X POST "$BASE_URL/orders/ORDER_ID/release-escrow"

# Response:
# { "message": "Escrow released. Funds paid to factory within 2 business days." }


##############################################
# STEP 6: Leave a review
##############################################

curl -X POST "$BASE_URL/orders/ORDER_ID/review" \
  -H "Content-Type: application/json" \
  -d '{
    "buyer_id": "buyer@example.com",
    "rating": 5,
    "quality_rating": 5,
    "communication_rating": 4,
    "accuracy_rating": 5,
    "comment": "Excellent quality, shipped on time."
  }'
Trust compounds: Each milestone with photo evidence triggers escrow release AND increases the factory's trust score. The next buyer benefits from the verified track record.
