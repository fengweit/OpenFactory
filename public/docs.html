Authentication

Endpoints marked with AUTH require a Bearer token. Register and login via the auth endpoints below.

POST /auth/register Register a new user ▶

Request Body

FieldTypeRequiredDescription
emailstringyesUser email
passwordstringyesPassword
role"buyer" | "factory"noUser role (default: buyer)
factory_idstringnoFactory ID (for factory role)

Example

curl -X POST ${BASE}/auth/register \ -H "Content-Type: application/json" \ -d '{"email":"buyer@example.com","password":"secret123","role":"buyer"}'

Response

{ "user_id": "usr_abc123", "email": "buyer@example.com", "role": "buyer", "token": "eyJhbGciOiJIUzI1NiIs..." }

Milestone Workflow

Every order follows this enforced milestone sequence. Each milestone must be reported in order — you cannot skip steps. Milestones marked with $ trigger automatic escrow releases.

📦
material_received
Raw materials arrived
→
⚙️ $
production_started
Production line active
→
🔬
qc_in_progress
Quality inspection
→
✅ $
qc_pass
QC passed
→
📋
ready_for_shipment
Packed & ready
→
🚢 $
shipped
In transit

Escrow release triggers: production_started releases a portion from deposit, qc_pass releases the production tranche, shipped releases the shipping tranche. Final release happens when the buyer calls confirm_receipt / POST /release-escrow.

Escrow State Machine

pending_deposit → deposit_held → production_released → final_released ↘ ↘ disputed disputed ↘ ↘ refunded refunded

At any stage, either party can raise a dispute which freezes the escrow. Platform admin resolves disputes with full/partial refund or rejection.

🔒
Escrow

Escrow flow: pending_deposit → deposit_held → production_released → final_released. Can enter disputed or refunded at any stage.

POST /orders/:id/escrow/lock Lock 30% deposit AUTH ▶

Buyer locks a 30% deposit. Transitions escrow from pending_deposit to deposit_held. Call right after order creation.

Example

curl -X POST ${BASE}/orders/ord-xyz789/escrow/lock \ -H "Authorization: Bearer YOUR_TOKEN"

Response

{ "order_id": "ord-xyz789", "deposit_locked": true, "deposit_amount_usd": 6300, "remaining_usd": 14700, "escrow_status": "deposit_held", "message": "Deposit locked. Factory can now begin production." }
POST /orders/:id/release-escrow Release final escrow on receipt ▶

Buyer confirms receipt. Validates milestone prerequisites (qc_pass required), transitions escrow to final_released, and triggers Stripe payout.

Example

curl -X POST ${BASE}/orders/ord-xyz789/release-escrow

Response

{ "order_id": "ord-xyz789", "escrow_status": "final_released", "message": "Escrow released. Funds will be paid to factory within 2 business days.", "escrow_event": { "from_status": "production_released", "to_status": "final_released", "triggered_by": "manual" } }

For AI Agents

OpenFactory exposes all 18 tools via the Model Context Protocol (MCP). Any MCP-compatible AI agent (Claude, GPT, custom agents) can connect and call tools directly — no REST integration needed.

MCP Server Connection

Add this to your Claude Desktop claude_desktop_config.json or any MCP client configuration:

claude_desktop_config.json

{
  "mcpServers": {
    "openfactory": {
      "command": "node",
      "args": ["dist/mcp/server.js"],
      "cwd": "/path/to/OpenFactory"
    }
  }
}

Or connect via npx directly:

npx -y @anthropic/mcp-client --server "node dist/mcp/server.js"

Agent Quickstart: Full Order Workflow

Here's the sequence an AI agent would follow to go from discovery to delivery:

Agent Tool Call Sequence

// 1. Find factories with available capacity
query_live_capacity({ category: "pcb_assembly", quantity: 5000 })

// 9. Verify QC passed

// 10. Confirm receipt & release final escrow
