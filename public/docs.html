Authentication

OpenFactory supports two authentication methods: JWT tokens for user sessions and API keys for programmatic / AI-agent access.

JWT Token Flow

For buyers, factories, and admins. Tokens expire after 7 days.

// 1. Register
POST /auth/register
{
  "email": "buyer@example.com",
  "password": "securepassword",
  "role": "buyer"           // buyer | factory | admin
}
// Response: { "user_id": "usr-...", "token": "eyJ..." }

// 2. Login
POST /auth/login
{
  "email": "buyer@example.com",
  "password": "securepassword"
}
// Response: { "user_id": "usr-...", "role": "buyer", "token": "eyJ..." }

// 3. Use in requests
Authorization: Bearer eyJ...

API Key Flow

For AI agents and partner integrations. Keys are prefixed ofk_ and rate-limited per minute.

// Generate key (admin only)
POST /admin/api-keys
{
  "partner_name": "my-agent",
  "permissions": ["GET /factories*", "POST /quotes"],
  "rate_limit_per_min": 60
}
// Response: { "key": "ofk_a1b2c3...", "id": "apk-..." }

// Use in requests
Authorization: Bearer ofk_a1b2c3...
Auth on endpoint cards: Auth = requires Bearer token (JWT or API key). No badge = public endpoint.

Base URL

// All endpoints are relative to your deployment origin
GET {origin}/factories/public
GET {origin}/health

In browser code, use window.location.origin to construct URLs. Do not hard-code localhost.

OpenAPI spec: The full machine-readable spec is served at /openapi.json — use it for code generation, SDK builders, or import into Postman/Insomnia.

MCP Server

OpenFactory exposes all capabilities as an MCP server, enabling AI agents to discover and call factory tools directly.

Connection Info

# Server name & version
name: openfactory
version: 0.3.0
transport: stdio

# Run the MCP server
npx ts-node src/mcp/server.ts

# Claude Desktop config (claude_desktop_config.json)
{
  "mcpServers": {
    "openfactory": {
      "command": "npx",
      "args": ["ts-node", "src/mcp/server.ts"]
    }
  }
}
MCP connection URL: For remote MCP over SSE/HTTP, connect to . The server auto-detects transport from the connection method.

MCP Tools

search_factories
Search verified GBA factories by category, MOQ, price tier, and rating. Returns ranked matches with capability data and certifications.
verify_factory_identity
Verify a factory's legal identity — USCC, legal representative, business license expiry, and link to gsxt.gov.cn registry.
factory_performance
Earned trust metrics from real transactional data: on-time delivery, lead time accuracy, QC pass rate, milestone responsiveness.
query_live_capacity
Real-time available capacity across all GBA factories. Returns who can fulfill your order right now with instant pricing. Sub-100ms.
get_instant_quote
Sub-second binding quote from a factory's pre-declared pricing rules. No waiting — computed instantly from tiered pricing. Valid 48h.
get_quote
Request a price quote from a specific factory. Returns unit price, total, lead time, MOQ, and a 7-day valid quote_id for ordering.
place_order
Place an order from an accepted quote. Payment held in escrow, released after buyer confirms receipt. Returns order_id.
track_order
Check production status — current status, full event history, and estimated ship date.
update_order_status
Update production milestone (factory-side). Valid: confirmed, in_production, qc, shipped, delivered. Supports photo attachments.
report_milestone
Report a production milestone with enforced ordering. Attach photos and notes for audit trail.
get_milestones
Full milestone timeline for an order — all milestones, photos, notes, and timestamps.
request_qc_inspection
Request third-party QC inspection (qima, sgs, bureau_veritas, manual). Requires production_started milestone.
get_qc_status
QC inspection status — provider, type, pass/fail result, and report URL.
lock_deposit
Lock 30% deposit on an order. Transitions escrow from pending_deposit to deposit_held.
check_escrow_status
Current escrow status with full audit trail and milestone chain.
confirm_receipt
Buyer confirms receipt. Validates qc_pass prerequisite, then transitions escrow to final_released.
raise_dispute
Raise a dispute on an order. Transitions escrow to disputed state. For quality, delivery, or payment issues.
submit_review
Buyer review with ratings (1-5) for overall, quality, communication, and accuracy. Updates factory trust score.
get_factory_reviews
All buyer reviews for a factory with summary statistics and aggregate averages.
get_analytics
Platform analytics: factory count, quote volume, order count, GMV, and per-factory activity.
