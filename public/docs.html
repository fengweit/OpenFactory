Authentication

Endpoints marked with AUTH require a Bearer token. Register and login via the auth endpoints below.

POST /auth/register Register a new user ▶

Request Body

FieldTypeRequiredDescription
emailstringyesUser email
passwordstringyesPassword
role"buyer" | "factory"noUser role (default: buyer)
factory_idstringnoFactory ID (for factory role)

Example

curl -X POST ${BASE}/auth/register \ -H "Content-Type: application/json" \ -d '{"email":"buyer@example.com","password":"secret123","role":"buyer"}'

Response

{ "user_id": "usr_abc123", "email": "buyer@example.com", "role": "buyer", "token": "eyJhbGciOiJIUzI1NiIs..." }
POST /auth/login Login and receive JWT token ▶

Request Body

FieldTypeRequiredDescription
emailstringyesUser email
passwordstringyesPassword

Example

curl -X POST ${BASE}/auth/login \ -H "Content-Type: application/json" \ -d '{"email":"buyer@example.com","password":"secret123"}'

Response

{ "user_id": "usr_abc123", "token": "eyJhbGciOiJIUzI1NiIs...", "role": "buyer" }

Identity Trust
GET /factories/:id/verify-identity Identity trust data for buyer agents ▶

Returns USCC (Unified Social Credit Code), legal representative, business license expiry, verification status, and a link to the Chinese national business registry (gsxt.gov.cn).

Path Parameters

FieldTypeRequiredDescription
idstringyesFactory ID (e.g. sz-001)

Example

curl ${BASE}/factories/sz-001/verify-identity

Response

{ "factory_id": "sz-001", "name": "Shenzhen Precision Electronics", "uscc": "91440300MA5EXAMPLE", "legal_rep": "Zhang Wei", "business_license_expiry": "2028-12-31", "verified": true, "verification_url": "https://gsxt.gov.cn/corp-query-...", "identity_complete": true }
POST /factories/:id/verify-uscc Validate and store USCC ▶

Validates a Unified Social Credit Code (18-char alphanumeric) and stores it against the factory record.

Request Body

FieldTypeRequiredDescription
usccstringyes18-char USCC (e.g. 91440300MA5EXAMPLE)

Example

curl -X POST ${BASE}/factories/sz-001/verify-uscc \ -H "Content-Type: application/json" \ -d '{"uscc":"91440300MA5EXAMPLE"}'

Response

{ "factory_id": "sz-001", "uscc": "91440300MA5EXAMPLE", "uscc_valid": true, "checksum_valid": true }
GET /factories/:id/trust-score Composite 0-100 trust score ▶

Returns a composite trust score (0-100) computed from identity completeness, verification status, transactional performance, and earned reputation.

Example

curl ${BASE}/factories/sz-001/trust-score

Response

{ "factory_id": "sz-001", "trust_score": 82, "components": { "identity_score": 25, "verification_score": 20, "performance_score": 22, "reputation_score": 15 }, "grade": "A" }
GET /factories/:id/performance Earned trust metrics from transactional data ▶

Returns on-time delivery rate, lead time accuracy, QC pass rate, milestone responsiveness, and total completed orders.

Example

curl ${BASE}/factories/sz-001/performance

Response

{ "factory_id": "sz-001", "total_orders": 47, "completed_orders": 42, "on_time_delivery_rate": 0.93, "avg_lead_time_accuracy": 0.88, "qc_pass_rate": 0.95, "milestone_responsiveness_hrs": 4.2 }

Discovery
GET /factories Search and filter verified factories ▶

Search verified Shenzhen factories by category, MOQ, price tier, rating, and trust score.

Query Parameters

FieldTypeRequiredDescription
categorystringnoelectronics_accessories | pcb_assembly | plastic_injection | metal_enclosure | cable_assembly
max_moqnumbernoMaximum acceptable MOQ
price_tierstringnobudget | mid | premium
min_ratingnumbernoMinimum factory rating (0-5)
verified_onlybooleannoOnly return verified factories
sortstringnoSort field (e.g. trust_score)
min_trust_scorenumbernoMinimum trust score (0-100)

Example

curl "${BASE}/factories?category=pcb_assembly&verified_only=true&min_trust_score=50"

Response

{ "factories": [ { "id": "sz-006", "name": "Nanshan IoT Systems", "categories": ["pcb_assembly", "electronics_accessories"], "moq": 200, "price_tier": "mid", "verified": true, "trust_score": 78, "identity_complete": true } ], "count": 1 }
GET /factories/:id Single factory with all fields ▶

Returns a single factory with all fields including identity data (USCC, legal rep, business license).

Example

curl ${BASE}/factories/sz-001

Response

{ "id": "sz-001", "name": "Shenzhen Precision Electronics", "name_zh": "深圳精密电子", "city": "Shenzhen", "district": "Nanshan", "categories": ["electronics_accessories", "pcb_assembly"], "moq": 300, "price_tier": "mid", "capacity_units_per_month": 50000, "verified": true, "trust_score": 82, "uscc": "91440300MA5EXAMPLE", "legal_rep": "Zhang Wei", "business_license_expiry": "2028-12-31" }
GET /capacity Live capacity query across all factories ▶

Query real-time manufacturing capacity across all GBA factories. Returns factories that can fulfill your order right now — with instant pricing, lead times, and confidence scores. Like AWS spot instances for manufacturing.

Query Parameters

FieldTypeRequiredDescription
categorystringyesProduct category
qtynumberyesRequired units
max_daysnumbernoMax acceptable lead time in days

Example

curl "${BASE}/capacity?category=pcb_assembly&qty=5000&max_days=30"

Response

{ "query": { "category": "pcb_assembly", "quantity": 5000, "max_days": 30 }, "count": 3, "results": [ { "factory_id": "sz-006", "factory_name": "Nanshan IoT Systems", "unit_price_usd": 4.20, "total_price_usd": 21000, "lead_time_days": 21, "available_units": 15000, "confidence": 0.95 } ], "computed_in_ms": 0 }
GET /factories/:id/instant-quote Sub-second binding quote ▶

Binding quote from pre-declared pricing rules. No waiting — price computed from tiered pricing + current capacity. Valid 48h.

Query Parameters

FieldTypeRequiredDescription
categorystringyesProduct category
qtynumberyesNumber of units

Example

curl "${BASE}/factories/sz-006/instant-quote?category=pcb_assembly&qty=5000"

Response

{ "factory_id": "sz-006", "factory_name": "Nanshan IoT Systems", "category": "pcb_assembly", "quantity": 5000, "unit_price_usd": 4.20, "total_price_usd": 21000, "lead_time_days": 21, "valid_until": "2025-04-15T00:00:00Z", "binding": true }
POST /quotes Request a price quote from a factory ▶

Request a price quote from a specific factory. Returns unit price, total, lead time, MOQ, and a 7-day valid quote_id. Triggers WeChat notification to the factory.

Request Body

FieldTypeRequiredDescription
factory_idstringyesFactory ID (e.g. sz-001)
product_descriptionstringyesWhat to manufacture
quantitynumberyesNumber of units
buyer_idstringnoBuyer identifier
target_price_usdnumbernoTarget unit price in USD
deadline_daysnumbernoDays until delivery needed

Example

curl -X POST ${BASE}/quotes \ -H "Content-Type: application/json" \ -d '{ "factory_id": "sz-006", "product_description": "Custom PCB for IoT sensor module", "quantity": 5000, "buyer_id": "buyer-42", "target_price_usd": 4.50 }'

Response

{ "quote_id": "q-abc123def", "factory_id": "sz-006", "unit_price_usd": 4.20, "total_price_usd": 21000, "lead_time_days": 21, "moq": 200, "valid_until": "2025-04-20T00:00:00Z" }
POST /rfq Broadcast RFQ to matching factories ▶

Broadcast a Request for Quotation to all matching factories. Creates individual quotes for each eligible factory.

Request Body

FieldTypeRequiredDescription
product_descriptionstringyesWhat to manufacture
quantitynumberyesNumber of units
categoriesstring[]yesProduct categories to match
target_price_usdnumbernoTarget unit price
max_lead_time_daysnumbernoMax acceptable lead time
buyer_idstringnoBuyer identifier

Example

curl -X POST ${BASE}/rfq \ -H "Content-Type: application/json" \ -d '{ "product_description": "USB-C cable assembly", "quantity": 10000, "categories": ["cable_assembly"], "target_price_usd": 1.20, "max_lead_time_days": 25 }'

Response (201)

{ "rfq_id": "rfq-abc123", "quotes_created": 4, "factories_matched": ["sz-003", "sz-007", "sz-012", "sz-015"], "status": "open" }
GET /rfq/:id Get quote responses for an RFQ ▶

Order Execution
POST /orders Place a manufacturing order AUTH ▶

Place a manufacturing order from an accepted quote. Payment held in escrow until buyer confirms receipt. Triggers WeChat notification, email confirmation, and Stripe escrow.

Request Body

FieldTypeRequiredDescription
quote_idstringyesquote_id from /quotes (valid 7 days)
buyer_idstringyesBuyer identifier

Example

curl -X POST ${BASE}/orders \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_TOKEN" \ -d '{"quote_id":"q-abc123def","buyer_id":"buyer-42"}'

Response

{ "order_id": "ord-xyz789", "factory_id": "sz-006", "status": "pending", "quantity": 5000, "unit_price_usd": 4.20, "total_price_usd": 21000, "escrow_held": true, "escrow_status": "pending_deposit", "estimated_ship_date": "2025-05-01" }

Returns full order details including milestones timeline and escrow event audit trail.

Example

Response

{ "order_id": "ord-xyz789", "status": "in_production", "factory_id": "sz-006", "total_price_usd": 21000, "escrow_status": "deposit_held", "milestones": [ { "milestone": "material_received", "created_at": "2025-04-05T10:00:00Z" }, { "milestone": "production_started", "created_at": "2025-04-07T08:30:00Z" } ], "escrow_events": [ { "from_status": "pending_deposit", "to_status": "deposit_held", "amount_usd": 6300 } ] }
PATCH /orders/:id/status Update order production status ▶

Update production status. Triggers shipping notification email when status is "shipped".

Request Body

FieldTypeRequiredDescription
status"confirmed" | "in_production" | "qc" | "shipped" | "delivered"yesNew production status
notestringnoNote (e.g. tracking number)
photo_urlsstring[]noPhoto URLs for this milestone

Example

curl -X PATCH ${BASE}/orders/ord-xyz789/status \ -H "Content-Type: application/json" \ -d '{"status":"shipped","note":"Tracking: SF1234567890"}'
POST /orders/:id/milestones Report a production milestone AUTH ▶

Factory reports a production milestone. Enforces ordering (e.g. production_started requires material_received). Only factory/admin roles. May trigger escrow transitions.

Request Body

FieldTypeRequiredDescription
milestone"material_received" | "production_started" | "qc_in_progress" | "qc_pass" | "qc_fail" | "ready_for_shipment" | "shipped"yesMilestone to record
photo_urlsstring[]noPhoto URLs documenting this milestone
notestringnoOptional note

Example

curl -X POST ${BASE}/orders/ord-xyz789/milestones \ -H "Content-Type: application/json" \ -H "Authorization: Bearer FACTORY_TOKEN" \ -d '{"milestone":"production_started","note":"Line 3 allocated"}'

Response (201)

{ "id": 5, "order_id": "ord-xyz789", "milestone": "production_started", "reported_by": "factory-user-01", "created_at": "2025-04-07T08:30:00Z", "photo_upload_url": "/orders/ord-xyz789/milestones/5/photos" }

Escrow

Escrow flow: pending_deposit → deposit_held → production_released → final_released. Can enter disputed or refunded at any stage.

POST /orders/:id/escrow/lock Lock 30% deposit AUTH ▶

Buyer locks a 30% deposit. Transitions escrow from pending_deposit to deposit_held. Call right after order creation.

Example

curl -X POST ${BASE}/orders/ord-xyz789/escrow/lock \ -H "Authorization: Bearer YOUR_TOKEN"

Response

{ "order_id": "ord-xyz789", "deposit_locked": true, "deposit_amount_usd": 6300, "remaining_usd": 14700, "escrow_status": "deposit_held", "message": "Deposit locked. Factory can now begin production." }
POST /orders/:id/release-escrow Release final escrow on receipt ▶

Buyer confirms receipt. Validates milestone prerequisites (qc_pass required), transitions escrow to final_released, and triggers Stripe payout.

Example

curl -X POST ${BASE}/orders/ord-xyz789/release-escrow

Response

{ "order_id": "ord-xyz789", "escrow_status": "final_released", "message": "Escrow released. Funds will be paid to factory within 2 business days.", "escrow_event": { "from_status": "production_released", "to_status": "final_released", "triggered_by": "manual" } }
GET /orders/:id/escrow-events Full escrow audit trail ▶

Example

curl ${BASE}/orders/ord-xyz789/escrow-events

Response

{ "order_id": "ord-xyz789", "escrow_events": [ { "id": 1, "from_status": "pending_deposit", "to_status": "deposit_held", "triggered_by": "manual", "amount_usd": 6300, "created_at": "2025-04-03T14:00:00Z" } ], "count": 1 }

Disputes
POST /orders/:id/dispute Raise a dispute on an order AUTH ▶

Buyer or factory raises a dispute. Transitions escrow to "disputed" state.

Request Body

FieldTypeRequiredDescription
raised_by"buyer" | "factory" | "platform"yesWho is raising the dispute
reasonstringyesReason for the dispute
evidence_urlsstring[]noEvidence URLs (photos, documents)

Example

curl -X POST ${BASE}/orders/ord-xyz789/dispute \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_TOKEN" \ -d '{ "raised_by": "buyer", "reason": "Units received with visible defects on 15% of batch", "evidence_urls": ["/uploads/evidence/defect1.jpg"] }'

Response (201)

{ "id": 1, "order_id": "ord-xyz789", "raised_by": "buyer", "reason": "Units received with visible defects on 15% of batch", "status": "open", "created_at": "2025-04-20T16:00:00Z" }
GET /orders/:id/dispute Get disputes for an order ▶

Example

curl ${BASE}/orders/ord-xyz789/dispute

Response

{ "order_id": "ord-xyz789", "disputes": [ { "id": 1, "raised_by": "buyer", "reason": "Defects on 15% of batch", "status": "open" } ], "count": 1 }

MCP Tools

18 MCP tools for AI agents via the Model Context Protocol. Connect your agent to OpenFactory and call these tools directly.

search_factories
Search verified Shenzhen factories by category, MOQ, price tier, and rating. Returns ranked matches with capability data and identity fields.
Params: category? max_moq? price_tier? min_rating? verified_only?
get_quote
Request a price quote from a specific factory. Returns unit price, total, lead time, and a 7-day valid quote_id.
Params: factory_id product_description quantity buyer_id? target_price_usd? deadline_days?
get_instant_quote
Sub-second binding quote from pre-declared pricing rules. No waiting. Valid 48h.
Params: factory_id category quantity
query_live_capacity
Real-time available manufacturing capacity across all GBA factories. Like AWS spot instances for manufacturing. Results in <100ms.
Params: category quantity max_days?
place_order
Place a manufacturing order from an accepted quote. Payment held in escrow until receipt confirmed.
Params: quote_id buyer_id
track_order
Check production status. Returns current status, full event history, and estimated ship date.
Params: order_id
update_order_status
Update production milestone (factory-side). Valid: confirmed | in_production | qc | shipped | delivered.
Params: order_id status note? photo_urls?
get_analytics
Platform analytics: factory count, quote volume, order count, GMV, and per-factory activity.
Params: none
verify_factory_identity
Verify a factory's legal identity. Returns USCC, legal rep, business license expiry, and gsxt.gov.cn link.
Params: factory_id
report_milestone
Report a production milestone. Enforces ordering (e.g. production_started requires material_received).
Params: order_id milestone photo_urls? note?
get_milestones
Full milestone timeline for an order including photos, notes, and timestamps.
Params: order_id
request_qc_inspection
Request third-party QC inspection. Makes escrow release credible. Providers: qima, sgs, bureau_veritas.
Params: order_id provider inspection_type
get_qc_status
QC inspection status. Returns provider, type, status, pass/fail result, and report URL.
Params: order_id
check_escrow_status
Current escrow status with full event audit trail and milestone chain.
Params: order_id
lock_deposit
Lock 30% deposit. Transitions escrow from pending_deposit to deposit_held.
Params: order_id
raise_dispute
Raise a dispute on an order. Transitions escrow to "disputed" state.
Params: order_id raised_by reason evidence_urls?
confirm_receipt
Buyer confirms receipt. Validates prerequisites then transitions escrow to final_released.
Params: order_id
factory_performance
Earned trust metrics from real transactional data. On-time delivery, QC pass rate, milestone responsiveness.
Params: factory_id
