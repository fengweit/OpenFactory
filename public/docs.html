The Trust Workflow

OpenFactory makes factory relationships machine-readable — every milestone is a POST, every QC result is a webhook, every escrow release is an API call.

The flow: search_factories → get_quote → place_order → report_milestone → request_qc_inspection → lock_deposit → confirm_receipt. Each step builds a verifiable trust chain backed by Stripe escrow and third-party QC.

🔑 Authentication

OpenFactory supports two authentication methods. Public endpoints (factory directory, instant quotes) work without any auth.

API Key (for AI agents)

Best for programmatic access. Pass your key in the x-api-key header on every request.

curl /factories \
  -H "x-api-key: your_api_key"

Request a key via POST /admin/api-keys or contact hello@openfactory.ai

Bearer JWT (for users)

For buyer/factory dashboards. Get a token via POST /auth/login, then pass as Bearer.

curl /factories \
  -H "Authorization: Bearer <jwt>"

Tokens issued by POST /auth/register or POST /auth/login

⚡ Core Endpoints

The three most important endpoints for integrating with OpenFactory.

GET /factories/public Search verified factories — no auth required ▼

Search verified GBA factories by category, city, or capability. No authentication needed.

Query Parameters
ParamTypeDescription
categorystring?e.g. pcb_assembly, plastic_injection, apparel_textiles
citystring?e.g. Shenzhen, Dongguan, Guangzhou
Example
GET /factories/public?category=pcb_assembly

// 200 OK
{
  "factories": [
    {
      "id": "sz-006",
      "name": "Nanshan IoT Systems",
      "verified": true,
      "categories": ["pcb_assembly"],
      "location": { "city": "Shenzhen", "district": "Nanshan" },
      "moq": 500,
      "rating": 4.6,
      "trust_score": 82
    }
  ]
}
GET /factories/{id}/instant-quote Binding quote in <100ms ▼

Get a binding price and lead time from a factory's pre-declared pricing rules. Responds in under 100ms — no emails, no waiting.

Parameters
ParamTypeDescription
idstringrequired — Factory ID (path param)
productstring?Product category slug
quantitynumber?Order quantity (must meet MOQ)
Example
GET /factories/sz-006/instant-quote?product=pcb_assembly&quantity=5000

// 200 OK — 36ms
{
  "factory": "Nanshan IoT Systems",
  "product": "pcb_assembly",
  "quantity": 5000,
  "unit_price_usd": 4.20,
  "total_usd": 21000,
  "lead_time_days": 21,
  "valid_for": "24h",
  "escrow": true,
  "response_ms": 36
}

🤖 MCP Server Connection

Connect your AI agent to OpenFactory via the Model Context Protocol. Your agent gets tools for searching factories, getting quotes, and placing orders — all callable in natural language.

1. Add to your MCP config
// claude_desktop_config.json
{
  "mcpServers": {
    "openfactory": {
      "command": "npx",
      "args": ["tsx", "src/mcp/server.ts"],
      "env": {
        "OPENFACTORY_API_URL": "https://your-host.com"
      }
    }
  }
}
2. Use tools via your agent
// Your agent can now call:
search_factories({ category: "pcb_assembly" })
get_instant_quote({ factory_id: "sz-006", quantity: 5000 })
check_capacity({ category: "pcb_assembly" })
place_order({ quote_id: "q-...", buyer_id: "buyer-001" })

For the full MCP integration guide, see the MCP integration page.

Escrow States
