Escrow State Machine

Every order carries an escrow_status. The platform enforces valid transitions — invalid moves return 400.

escrow-states.txt 
  ┌─── Happy Path ──────────────────────────────────────────────────┐

  pending_deposit ───→ deposit_held ───→ production_released ───→ final_released ✓
      POST /escrow/lock      milestone: production_started       POST /release-escrow
      (buyer locks 30%)       (auto-transition)                    (requires qc_pass)

  ┌─── Dispute Paths ───────────────────────────────────────────────┐

  deposit_held ───→ disputed ───→ refunded ✓          (full/partial refund)
  production_released ───→ disputed ───→ final_released ✓  (dispute rejected)

  ┌─── Cancellation ────────────────────────────────────────────────┐

  pending_deposit ───→ refunded ✓   (cancelled before deposit)
  deposit_held ───→ refunded ✓      (refund before production)

Transition Table

From Status Allowed Transitions Trigger
pending_deposit deposit_held, refunded POST /orders/:id/escrow/lock, or cancel
deposit_held production_released, disputed, refunded Auto on production_started milestone, dispute, or refund
production_released final_released, disputed POST /orders/:id/release-escrow (requires qc_pass), or dispute
final_released TERMINAL Payment complete — no further transitions
disputed refunded, production_released, final_released POST /disputes/:id/resolve (admin)
refunded TERMINAL Funds returned — no further transitions

Quickstart: Full Flow

Six API calls — registration to escrow release. Replace $BASE with your server URL.

quickstart.sh 
# 1. Register and get a JWT token
TOKEN=$(curl -s -X POST $BASE/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"buyer@acme.com","password":"s3cret","role":"buyer"}' \
  | jq -r '.token')

# 2. Browse verified factories
curl -s "$BASE/factories?category=pcb_assembly&verified_only=true" \
  -H "Authorization: Bearer $TOKEN" | jq '.[0]'

# 3. Request a quote
QUOTE_ID=$(curl -s -X POST $BASE/quotes \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"factory_id":"sz-006","product_description":"4-layer PCB 50x80mm","quantity":5000}' \
  | jq -r '.quote_id')

# 4. Place order (escrow_status → pending_deposit)
ORDER_ID=$(curl -s -X POST $BASE/orders \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"quote_id\":\"$QUOTE_ID\"}" \
  | jq -r '.order_id')

# 5. Factory reports a production milestone
curl -s -X POST "$BASE/orders/$ORDER_ID/milestones" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"milestone":"production_started","note":"PCB etching begun"}'

# 6. Buyer releases final escrow (requires qc_pass milestone)
curl -s -X POST "$BASE/orders/$ORDER_ID/release-escrow" \
  -H "Authorization: Bearer $TOKEN"

# Done — escrow_status: final_released, factory gets paid
