OpenFactory API Reference

REST endpoints and MCP tools to search, quote, order, and track manufacturing across 847 verified Shenzhen factories.

🔑 Authentication

Endpoints marked AUTH require a JWT bearer token or API key via Authorization header or x-api-key. Endpoints marked PUBLIC need no credentials.

POST /auth/register Register new user PUBLIC ▶
Request Body
{ "email": "buyer@example.com", "password": "securepass", "role": "buyer" }
Parameters
NameTypeDescription
emailstringUser email address
passwordstringAccount password
rolestring optional"buyer" or "factory"
factory_idstring optionalLink account to factory (factory role)
Response
{ "user": { "id": "usr-001", "email": "buyer@example.com", "role": "buyer" }, "token": "eyJhbG..." }
Example
curl -X POST $BASE/auth/register \ -H "Content-Type: application/json" \ -d '{"email":"buyer@example.com","password":"securepass","role":"buyer"}'
POST /auth/login Login & get JWT token PUBLIC ▶
Request Body
{ "email": "buyer@example.com", "password": "securepass" }
Response
{ "user": { "id": "usr-001", "email": "...", "role": "buyer" }, "token": "eyJhbG..." }
Example
curl -X POST $BASE/auth/login \ -H "Content-Type: application/json" \ -d '{"email":"buyer@example.com","password":"securepass"}'

Description

Returns USCC (Unified Social Credit Code), legal representative, business license expiry, and verification status for buyer agents to validate factory legitimacy.

Response

Request Body

Milestone Values
production_started | qc_pass | shipped
Example
curl -X POST $BASE/orders/ord-456/escrow/release \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ -d '{"milestone":"qc_pass"}'

Description

Buyer confirms receipt → final escrow release to factory. Requires qc_pass milestone.

Response

curl -X POST $BASE/orders/ord-456/release-escrow

Escrow State Machine

🤖 For AI Agents

Connect via MCP (Model Context Protocol) to access all 18 manufacturing tools from your AI agent.

MCP Server: openfactory v0.2.0

Turn verified Shenzhen factories into callable tools. Search, quote, order, and track manufacturing — all from your AI agent.

Connection (stdio transport)
npx openfactory-mcp
Or connect via SSE

All 18 Tools

search_factories
Search 847 verified Shenzhen factories by category, MOQ, price tier, and rating. Returns ranked matches with capability data, certifications, WeChat contact, and identity fields.
get_quote
Request a price quote from a specific factory. Returns unit price, total price, lead time, MOQ, and a 7-day valid quote_id for ordering.
place_order
Place a manufacturing order from an accepted quote. Payment is held in escrow and only released after buyer confirms receipt.
track_order
Check production status. Returns current status, full event history (order_placed → confirmed → in_production → qc → shipped → delivered), and estimated ship date.
update_order_status
Update the production milestone of an order (factory-side). Valid statuses: confirmed | in_production | qc | shipped | delivered.
get_analytics
Get platform analytics: factory count, quote volume, order count, GMV, and per-factory quote activity.
get_instant_quote
Sub-second binding quote from pre-declared pricing rules. No waiting — price computed instantly from tiered pricing + current capacity. Valid 48h.
query_live_capacity
Query real-time available manufacturing capacity across all GBA factories. Returns only factories that can fulfill your order RIGHT NOW — with instant pricing and lead times. Results in <100ms.
verify_factory_identity
Verify a factory's legal identity. Returns USCC, legal representative, business license expiry, verification status, and a link to gsxt.gov.cn.
report_milestone
Report a production milestone for an order. Enforces milestone ordering (e.g. production_started requires material_received). Attach photos and notes for audit trail.
get_milestones
Get the full milestone timeline for an order, including all production milestones, photos, notes, and timestamps.
request_qc_inspection
Request third-party QC inspection. Providers: qima, sgs, bureau_veritas. The missing link between self-reported milestones and independently verified quality.
get_qc_status
Get QC inspection status. Returns all inspection requests with provider, type, status, pass/fail result, and report URL.
check_escrow_status
Check current escrow status for an order. Returns escrow_status, full escrow event audit trail, and milestone chain.
lock_deposit
Lock 30% deposit on an order. Transitions escrow from pending_deposit to deposit_held. Call right after place_order to commit buyer intent.
raise_dispute
Raise a dispute on an order. Transitions escrow to 'disputed' state. Use when buyer or factory has a quality, delivery, or payment issue.
confirm_receipt
Buyer confirms receipt of goods. Validates milestone prerequisites (qc_pass required) then transitions escrow to final_released, releasing payment to factory.
factory_performance
Earned trust metrics from real transactional data: on-time delivery rate, lead time accuracy, QC pass rate, milestone responsiveness, and completed orders.
